# 📚 주목 논문

## 1. Less Context, Better Agents: Efficient Context Engineering for Long-Horizon Tool-Using LLM Agents
- **arXiv:** [2606.10209](https://arxiv.org/abs/2606.10209) · cs.AI/cs.LG/cs.SE · 제출 2026-06-08 (17p·3fig·8tbl)
- **저자:** Abhilasha Lodha, Mahsa Pahlavikhah Varnosfaderani, Abir Chakraborty, Abhinav Mithal
- **핵심 기여:** 엔터프라이즈 시스템의 *장황한 툴 응답*이 컨텍스트 오버플로와 과도한 추론 비용을 유발해 대규모 태스크 완수를 막는다는 문제를, **외부 스토어·리트리버 없이** 푸는 미니멀 처방.
- **방법:** long-horizon 툴 사용 워크플로에서 (a) 가벼운 **recency window**(최근 N턴)와 (b) **컴팩트 러닝 요약** 조합만으로 충분함을 실험적으로 보임. RAG 메모리 인프라를 깔기 전의 강한 베이스라인.
- 🛠 **빌더 관점:** 메모리·벡터DB부터 붙이지 말고, "최근 N턴 + 러닝 요약"을 먼저 깔고 *그 위에서* 무엇이 더 필요한지 측정하라는 실전 순서.
- ⚠️ **한계:** 엔터프라이즈 툴 워크플로 중심 평가 → 도메인 일반화는 추가 검증 필요.

## 2. Less Context, More Accuracy: A Bi-Temporal Memory Engine (Engram)
- **arXiv:** [2606.09900](https://arxiv.org/abs/2606.09900) · cs.AI · 제출 2026-06-05
- **저자:** Liuyin Wang (Independent Researcher)
- **핵심 기여:** 세션을 넘나들며 에이전트가 "잊어버리는" 문제에서, 전체 히스토리 재생은 비싸고 distractor가 쌓일수록 정확도가 떨어진다는 점을 짚고 — **lean 검색 컨텍스트가 full-context 베이스라인을 정확도로 이긴다**고 주장.
- **방법:** **Engram** = bi-temporal 데이터 모델 기반 듀얼-프로세스 메모리. ① LLM을 임계 경로에서 뺀 빠른 write path로 무손실 episode를 append, ② 비동기 consolidation path로 atomic fact를 추출. 재현 가능한 하네스·코드 공개: [github.com/ly-wang19/engram](https://github.com/ly-wang19/engram).
- 🛠 **빌더 관점:** "이벤트 발생 시각 vs 기록 시각"을 분리하는 **bi-temporal 메모리**는 시간 의존 질의(언제 무엇이 사실이었나)를 다루는 에이전트에 곧장 이식 가능.
- ⚠️ **한계:** 단독 저자·초기 단계 repo(현재 소규모). 벤치 수치는 공개 하네스로 직접 재현 권장.

![lean 컨텍스트 패턴 다이어그램: 전체 히스토리 주입과 recency window+요약+bi-temporal 검색 비교](/images/ai-news-2026-06-19/lean-context.svg)

*두 논문이 가리키는 같은 결론 — 다 넣는 것보다 골라 넣는 쪽이 정확도까지 가져간다.*

# 🧑‍💻 Codex & Claude Code 실전 팁

### 🟩 Codex — `/import`로 Claude Code 설정 그대로 옮기기
- **카테고리:** Workflow / Migration · 출처: [Codex 업데이트](https://releasebot.io/updates/openai/codex)
- 💡 **핵심:** Codex CLI가 `/import`로 Claude Code의 setup·프로젝트 설정·최근 채팅을 **선택적으로** 가져옵니다. 처음부터 다시 세팅할 필요 없음.
- 📝 **방법:**
```bash
# Codex CLI 안에서
/import          # setup · project config · recent chats 중 선택 임포트
/archive         # 끝난 스레드 보관 (resume/fork 잠금)
codex delete     # 영구 삭제 (서브에이전트까지 정리)
```
- 🎯 **언제:** Claude Code → Codex로 이주하거나 두 툴을 병행하며 컨텍스트를 공유하고 싶을 때.
- ⚠️ **주의:** 현재 alpha 채널(vrust-v0.141.0-alpha.4) 기능 — 프로덕션 핵심 설정은 임포트 후 검증.
- ⭐ **추천:** 툴 갈아타며 매번 설정을 다시 짜던 사람에게 즉효.

### 🟦 Claude Code — `Tool(param:value)` 권한 룰로 서브에이전트 비용 캡
- **카테고리:** Subagent / Permission · 출처: [Claude Code Changelog](https://code.claude.com/docs/en/changelog) (v2.1.178+)
- 💡 **핵심:** 권한 룰을 파라미터 단위로 걸 수 있습니다. 예를 들어 비싼 Opus 서브에이전트 자동 생성을 막아 토큰 폭주를 차단.
- 📝 **방법:**
```text
# settings.json permissions.deny 등에
Agent(model:opus)        # Opus 서브에이전트 차단 → 기본 모델로 강제
# 즉석 토글이 필요하면
/config thinking=false   # 프롬프트에서 바로 설정 변경
```
- 🎯 **언제:** 서브에이전트가 무분별하게 상위 모델을 호출해 비용이 튈 때, 또는 CI/배치에서 모델을 못박고 싶을 때.
- ⚠️ **주의:** 차단하면 해당 태스크 품질이 떨어질 수 있으니 *비용 vs 정확도*를 보고 결정.
- ⭐ **추천:** 멀티 서브에이전트를 굴리는 팀이라면 비용 가드레일로 기본 적용.

### 🟪 공통 — RAG 깔기 전에 "요약 베이스라인"부터
- **카테고리:** Context / Eval · 출처: [arXiv 2606.10209](https://arxiv.org/abs/2606.10209) · [InfoQ: 토큰 절감](https://www.infoq.com/news/2026/05/github-agentic-token-savings/)
- 💡 **핵심:** 오늘 논문의 실전 함의 — 메모리·벡터DB부터 붙이지 말고 **recency window + 러닝 요약**을 먼저 베이스라인으로 두세요.
- 📝 **방법:** 압축은 *명시적·주기적*일 때만 효과. 수동(passive) 프롬프트는 약 6% 절감에 정확도 저하; 10~15 툴콜마다 압축을 명령하면 정확도를 지키며 토큰을 22%대까지 절감(SWE-bench 보고치).
- 🎯 **언제:** 툴 응답이 비대해 컨텍스트가 자꾸 터지는 에이전트 루프.
- ⚠️ **주의:** "압축하라"를 모델 선의에 맡기지 말고 스케줄로 강제할 것.
- ⭐ **추천:** 메모리 스택 투자 전 ROI를 가르는 핵심 베이스라인.
