# 🔥 오늘의 핵심

오늘의 축은 **"값싼 오픈웨이트가 프런티어를 추격하고, 에이전트는 신뢰를 설계로 얻는다"**이다. Moonshot **Kimi K3**(2.8T MoE)가 **7/27 오픈웨이트 공개**를 예고하며 Artificial Analysis 랭킹 3위에 올랐다. 연구 두 편은 각각 에이전트 **실패를 폐루프로 디버깅**(AgentDebugX)하고 **프롬프트 주입을 권한 분리로 방어**(Twin Agent)한다. 오늘 할 일: 에이전트 루프에 "관측→귀인→복구" 디버깅 규율을 넣고, 신뢰 불가 입력은 실행 권한에서 떼어내라.

# 🤖 모델 & 릴리즈 레이더

- **Kimi K3 (Moonshot AI) — 사상 최대 오픈웨이트, 공개 임박.** 2.8T 파라미터 **sparse MoE**(요청당 896개 중 16개 전문가 활성), **1M 컨텍스트**, 멀티모달. 7/16 API 출시 후 **7/27 전체 웨이트 공개** 예고. 가격 **입력 $3 / 출력 $15 per 1M**(캐시 히트 입력 $0.30). Artificial Analysis 리더보드 **3위**(1위 Claude Fable 5, 2위 GPT-5.6 Sol)이지만 **블라인드 프런트엔드 코딩(Frontend Code Arena)에서는 Fable 5보다 선호**됐고 Terminal-Bench 2.1 **88.3**. → **에이전트 빌더 시사점**: 코딩·롱호라이즌 툴콜 루프의 저비용 티어 후보로 A/B 값어치가 크다. 특히 7/27 이후 **자가 호스팅**이 열리면 데이터·레이트리밋 통제가 필요한 워크로드에 대안이 된다. 출처: [tomshardware](https://www.tomshardware.com/tech-industry/artificial-intelligence/moonshot-releases-2-8-trillion-parameter-kimi-k3) · [mlq.ai](https://mlq.ai/news/moonshot-ai-releases-kimi-k3-a-28-trillion-parameter-open-weight-model-rivaling-top-us-systems/)
- **Claude Code v2.1.218 (7/22) — `/code-review`가 백그라운드 서브에이전트로.** 리뷰 작업이 메인 대화창을 채우지 않고 별도 세션에서 돌며, 스택된 슬래시 커맨드를 리뷰 대상으로 잡는다. → **시사점**: 리뷰를 "머지 게이트"로 상시 돌려도 컨텍스트 오염이 없다. 출처: [code.claude.com/changelog](https://code.claude.com/docs/en/changelog)
- **Codex — Record & Replay·Sites 프리뷰.** macOS에서 **시연한 워크플로를 재사용 가능한 skill로 녹화**하고, Sites(프리뷰)로 대시보드·내부툴을 배포하며, **Windows에서도 Computer Use** 지원. → **시사점**: 반복 GUI 작업을 skill로 굳혀 에이전트 파이프라인에 끼울 수 있다. 출처: [developers.openai.com/codex/changelog](https://developers.openai.com/codex/changelog)

# 📚 주목 논문

## 1) AgentDebugX — 에이전트 실패를 '폐루프'로 디버깅
- **저자**: Kunlun Zhu, Xuyan Ye, Zhiguang Han 외 (arXiv `2607.18754`, cs.AI, 2026-07-21)
- **핵심**: 에이전트 실패는 **에러가 드러난 스텝이 원인 스텝이 아니기** 때문에 디버깅이 어렵다. 기존 관측 툴은 트레이스를 재생만 할 뿐 근본원인·복구로 이어주지 못한다.
- **방법(요약)**: 디버깅을 **Detect → Attribute → Recover → Rerun** 폐루프로 구성. 핵심 모듈 `DeepDebug`가 전역 궤적 이해·구조 유도 조사·교차검증으로 멀티턴 근본원인 진단을 수행한다.
- **결과(검증됨)**: Who&When 벤치 **엄격 귀인 정확도** qwen3.5-9b에서 **28.8%**(최고 단일패스 베이스라인 21.7%). GAIA에서 실패 태스크 **73건 중 13건 복구**.
- **🛠 agent-builder 시사점**: 관측을 "트레이스 뷰어"에서 멈추지 말고 **귀인+복구+재실행**까지 자동화하라. 어제 CodeRescue가 '복구를 언제 승격할지'였다면, 오늘은 '무엇이 실패를 유발했는지'를 짚는 도구다.
- **한계**: 엄격 귀인 정확도 자체가 28.8%로 아직 낮다 — 사람 검토 없는 완전 자동 복구는 시기상조.
- 링크: [arxiv.org/abs/2607.18754](https://arxiv.org/abs/2607.18754)

![AgentDebugX 폐루프 다이어그램. Detect→Attribute→Recover→Rerun 순환과 DeepDebug 멀티턴 진단, 그리고 엄격 귀인 정확도 막대(DeepDebug 28.8% vs 최고 단일패스 21.7%)](/images/ai-news-2026-07-24/agentdebugx-loop.svg)
*'터진 곳' 말고 '터뜨린 곳'을 찾는 게 핵심. (자체 제작 SVG, 값 출처 arXiv 2607.18754)*

## 2) Twin Agent — 권한 분리로 프롬프트 주입 막기
- **저자**: Zhanhao Hu, Dennis Jacob, Zhaorun Chen, Bo Li, David Wagner 외 (arXiv `2607.19595`, cs.CR, 2026-07-21)
- **핵심**: 신뢰 불가 컨텍스트가 다운스트림 추론·툴 사용을 조작하는 **프롬프트 주입**을 막는 일반 설계 패턴.
- **방법(요약)**: 거의 대칭인 두 에이전트 — 신뢰 불가 정보를 살피는 **Explore Agent**와 특권 액션을 실행하는 **Safe Agent**. Explore는 Safe의 현재 컨텍스트로 조건화되어 **다음 액션에 대한 compact hint만** Safe에 전달한다(residual 압축).
- **🛠 agent-builder 시사점**: "정보 흐름을 좁히면 유틸리티가 준다"는 기존 secure-by-design의 딜레마를, **컨텍스트 잔차(residual)만 전달**해 완화한다. 웹 크롤·문서 요약·툴 응답을 실행 권한과 분리하는 실전 템플릿으로 참고.
- **한계**: 태스크별 튜닝 없이도 동작하도록 설계됐지만, 두 에이전트 왕복 비용이 추가된다 — 지연에 민감한 루프는 측정 필요.
- 링크: [arxiv.org/abs/2607.19595](https://arxiv.org/abs/2607.19595)

![Twin Agent 아키텍처. 신뢰 불가 컨텍스트를 Explore Agent가 조사하고 compact hint만 Safe Agent에 전달해, 주입 지시가 특권 실행에 직접 흘러들지 못하게 한다](/images/ai-news-2026-07-24/twin-agent.svg)
*신뢰 경계를 '설계 패턴'으로. (자체 제작 SVG, 개념 출처 arXiv 2607.19595)*

# 🛠 오픈소스 스포트라이트

## OpenClaw — GitHub 최다 스타 개인 AI 어시스턴트
- **repo**: [github.com/openclaw/openclaw](https://github.com/openclaw/openclaw)
- **스타(웹 기준)**: ~346k★ (이번 세션 API 미검증, 웹 출처) — 5개월 만에 GitHub 최다 스타 소프트웨어 repo

## Agno — 자기 클라우드에서 굴리는 에이전트 플랫폼
- **repo**: [github.com/agno-agi/agno](https://github.com/agno-agi/agno)
- **언어**: Python · **스타(웹 기준)**: ~41k★ (이번 세션 API 미검증, 웹 출처) · Apache-2.0

- **에이전트 개발자 활용처**: 매니지드 SaaS 대신 **자기 클라우드**에 에이전트를 올려야 하는 팀의 베이스. 권한·휴먼리뷰가 1급 개념이라 규제 워크로드에 맞다.

# 🧑‍💻 Codex & Claude Code 실전 팁

**🟩 Codex · category: Workflow**
- 출처: [developers.openai.com/codex/changelog](https://developers.openai.com/codex/changelog)
- 💡 **핵심**: 반복되는 GUI/터미널 작업을 매번 프롬프트로 지시하지 말고, **Record & Replay로 한 번 시연해 skill로 굳혀라**(macOS).
- 📝 **how-to**: 앱 실행에서 워크플로를 실제로 한 번 수행 → 데모를 저장하면 재사용 skill이 생성됨 → 이후 에이전트가 그 skill을 호출.
- 🎯 **언제**: 배포·리포트 생성처럼 손이 정해진 반복 작업.
- ⚠️ **주의**: 시연에 자격증명·토큰 입력이 섞이면 skill에 흔적이 남을 수 있으니 민감정보는 env/시크릿 매니저로 분리.
- ⭐ **추천**: 손이 기억하는 반복작업 1개를 오늘 녹화해보라.

**🟦 Claude Code · category: Review**
- 출처: [code.claude.com/docs/en/changelog](https://code.claude.com/docs/en/changelog) (v2.1.218)
- 💡 **핵심**: `/code-review`가 **백그라운드 서브에이전트**로 돌아, 리뷰가 메인 대화창을 오염시키지 않는다.
- 📝 **how-to**:
```bash
# 작업을 이어가면서 리뷰를 백그라운드로
/code-review
# 리뷰가 도는 동안 메인 세션은 계속 코딩/편집 가능
# 진행/결과는 claude agents 로 확인
claude agents
```
- 🎯 **언제**: 큰 diff를 머지 전 상시 검토 게이트로 세울 때.
- ⚠️ **주의**: 2.1.215부터 `/verify`·`/code-review`는 **자동 실행되지 않는다** — 명시적으로 호출할 것.
- ⭐ **추천**: PR 올리기 전 `/code-review` → 지적 반영 → 머지의 2단계 루틴화.

**🟪 공통 · category: Debugging**
- 출처: [arxiv.org/abs/2607.18754](https://arxiv.org/abs/2607.18754) (AgentDebugX)
- 💡 **핵심**: 에이전트 실패를 로그 재생으로만 보지 말고 **Detect → Attribute → Recover → Rerun 폐루프**로 처리하라. 에러가 *드러난* 스텝이 아니라 *유발한* 스텝을 짚는 게 관건.
- 📝 **how-to**: (1) 실패 신호 감지 → (2) 전체 궤적을 놓고 근본원인 스텝 귀인 → (3) 그 스텝에 타깃 복구 적용 → (4) 재실행으로 검증. 관측 툴을 "뷰어"에서 멈추지 말 것.
- 🎯 **언제**: 멀티스텝 툴콜 에이전트가 후반부에서 조용히 틀어질 때.
- ⚠️ **주의**: 현 시점 엄격 귀인 정확도는 ~28.8%로 낮다 — 완전 자동 복구보다 **사람 검토 보조**로 도입.
- ⭐ **추천**: 실패 트레이스에 "원인 스텝 추정" 필드를 하나 붙여 습관화.
