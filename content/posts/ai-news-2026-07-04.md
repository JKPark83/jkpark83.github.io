# 🔥 오늘의 핵심

프런티어 클로즈드 모델은 여전히 **잠겨 있다** — **GPT-5.6 Sol**은 美 정부 승인 ~20개 조직 뒤, **Gemini 3.5 Pro**는 6월 GA에서 또 **7월로 재연기**돼 아직 Vertex 프리뷰다. 그 틈에 **Meituan이 LongCat-2.0(1.6T MoE·1M 컨텍스트)을 MIT로 오픈**했고, SWE-bench Pro에서 GPT-5.5를 근소하게 앞선다. 그래서 오늘 개발자가 할 일은 명확하다 — **에이전트 코드는 그대로 두고 게이트웨이(OpenRouter/LiteLLM)로 LongCat-2.0을 끼워, 내 태스크에서 클로즈드 모델과 A/B** 해보라.

# 🧑‍💻 Codex & Claude Code 실전 팁

## 🟩 Codex — Provider / Workflow
- 소스: [Codex Configuration Reference](https://developers.openai.com/codex/config-reference) · [OpenRouter × Codex CLI](https://openrouter.ai/docs/cookbook/coding-agents/codex-cli)
- 💡 핵심: Codex는 **커스텀 프로바이더**를 지원한다. `config.toml`에 OpenRouter를 등록하면 GPT 계열이 아닌 **LongCat-2.0 같은 오픈모델을 그대로 코딩 에이전트로** 굴릴 수 있다.
- 📝 방법:

```toml
# ~/.codex/config.toml
[model_providers.openrouter]
name = "OpenRouter"
base_url = "https://openrouter.ai/api/v1"
env_key = "OPENROUTER_API_KEY"
wire_api = "responses"   # 2026-02 이후 chat 값 제거됨 — 반드시 responses

[profiles.longcat]
model_provider = "openrouter"
model = "meituan/longcat-2.0"
```

```bash
# 프로필로 전환하거나, 한 번만 오버라이드
codex --profile longcat
codex -c model_provider='"openrouter"' -c model='"meituan/longcat-2.0"'
```

- 🎯 언제: 클로즈드 모델 비용이 부담되거나, 폐쇄망/규제로 특정 벤더를 못 쓸 때. 예약 ID(`openai`/`ollama`/`lmstudio`)는 재사용 불가.
- ⚠️ 주의: `wire_api = "responses"`가 아니면 기동 시 에러. OpenRouter 모델 ID는 실제 카탈로그에서 확인.
- ⭐ 추천: 모델 비용 비교/폴백을 실험 중이라면 강력 추천. 단일 벤더 고정이라면 불필요.

## 🟦 Claude Code — Workflow / Debugging
- 소스: [Claude Code 변경로그](https://code.claude.com/docs/en/changelog)
- 💡 핵심: 세션을 껐다 켜지 않고도 **작업 디렉터리를 옮기고(`/cd`)**, 문제 진단 시 **모든 커스터마이징을 끈 채(`--safe-mode`)** 깨끗한 상태로 재현한다.
- 📝 방법:

```bash
# 캐시를 버리지 않고 작업 디렉터리 이동 (재시작 시 캐시 재생성 비용 발생하던 문제 해소)
/cd ../another-service

# CLAUDE.md·플러그인·스킬·훅·MCP를 모두 비활성화하고 기동 (원인 격리용)
claude --safe-mode
# 또는
CLAUDE_CODE_SAFE_MODE=1 claude
```

- 🎯 언제: 모노레포에서 서비스 간 이동이 잦을 때(`/cd`), "훅/플러그인 때문에 이상한가?"를 가릴 때(`--safe-mode`).
- ⚠️ 주의: `--safe-mode`는 CLAUDE.md·MCP까지 끄므로 그 상태의 동작이 "정상"이 아님을 기억. 원인만 격리하고 평소 모드로 복귀.
- ⭐ 추천: 원인 불명 버그를 만나면 첫 수로 `--safe-mode` 재현부터.

## 🟪 공통 — Eval / Routing
- 소스: [LiteLLM](https://github.com/BerriAI/litellm)
- 💡 핵심: 오픈모델로 "갈아탈지"는 **감이 아니라 게이트로** 결정한다. 게이트웨이에 후보 모델을 등록하고 **같은 태스크·같은 프롬프트**로 Δ를 재라.
- 📝 방법: LiteLLM 프록시에 `router` 설정으로 후보군을 묶고, 각 요청에 태그를 달아 **정확도·툴콜 성공률·출력 토큰·$/요청**을 함께 로깅 → 대시보드에서 Δ 비교.
- 🎯 언제: LongCat-2.0(또는 임의 오픈모델)을 프로덕션 코딩 에이전트에 넣기 직전.
- ⚠️ 주의: 단발 샘플로 판단 금지. 실패 케이스(리트라이·컨텍스트 초과)까지 포함한 분포로 보라.
- ⭐ 추천: 비용 절감을 노린다면 필수 절차. Δ가 (+)이거나 비용이 크게 낮을 때만 승격.
