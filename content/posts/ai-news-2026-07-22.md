# 📚 주목 논문

## 1) TRIM — 에이전트 코드슬롭을 '궤적 최소화'로 줄이기
- **저자**: Alex Mathai, Shobini Iyer, Aleksandr Nogikh, Petros Maniatis, Franjo Ivancic, Junfeng Yang, Baishakhi Ray (arXiv `2607.18161`, cs.SE, 2026-07-20 제출)
- **핵심**: 코딩 에이전트가 뱉는 코드가 사람 구현보다 길고 장황한 원인을 **에이전트의 탐색 과정 자체**로 지목. 통과 해답으로 수렴하는 동안 쌓인 **투기적 편집·폐기된 가설·임시 변경**이 최종 패치에 잔존한다.
- **방법(요약)**: 불필요한 코드 조각을 직접 노리지 않고 **에이전트 궤적(trajectory)을 최소화**한다(Trajectory-guided Redundancy Identification and Minimization). 즉 "결과물"이 아니라 "만드는 과정"을 줄인다.
- **결과(검증됨)**: 여러 agentic 스캐폴드에서 **코드슬롭 17.9~32.9% 감소**, **성능 저하 거의 없음**. 검증 비용은 Delta Debugging 같은 알고리즘 베이스라인의 **약 절반**.
- **🛠 agent-builder 시사점**: 코딩 에이전트를 CI에 붙였다면, 최종 diff를 그대로 머지하지 말고 **"궤적 잔여물 제거" 후처리**를 하나 넣어라. 리뷰 부담과 유지보수 비용이 바로 줄어든다.
- **한계**: 벤치가 agentic 코드 생성 시나리오 중심이라, 팀 코드베이스별 룰과 결합했을 때의 효과는 별도 측정 필요.
- 링크: [arxiv.org/abs/2607.18161](https://arxiv.org/abs/2607.18161)

![TRIM 방법 다이어그램. 에이전트 탐색에서 투기적 편집·폐기 가설·임시 변경이 쌓여 코드슬롭이 되고, TRIM은 궤적 자체를 최소화해 코드슬롭 17.9~32.9%를 줄인다](/images/ai-news-2026-07-22/trim-flow.svg)
*코드가 아니라 '과정'을 줄이는 게 TRIM의 아이디어. (자체 제작 SVG, 값 출처 arXiv 2607.18161)*

## 2) Adaptive Adversaries — 적응형 멀티턴 에이전트 보안 벤치
- **저자**: Devina Jain, David Hartmann, Chuan Li (arXiv `2607.18063`, cs.CR, 2026-07-20 제출)
- **핵심**: 대부분의 안전 벤치는 **고정된 공격 풀**로 방어자를 평가한다. 이 논문은 **직전 응답을 보고 라운드마다 전략을 바꾸는 자율 공격 LLM**으로 21개 시나리오를 만든다.
- **결과(검증됨)**: 첫 턴만 채점하면 공격성공률(ASR) **0~1%**인데, **15라운드 적응 공격을 허용하면 5.4~14.0%**로 뛴다. 프런티어 공격자 3종을 **풀링하면 최상 단일 공격자 대비 고유 성공 공격을 1.4~2.2배** 더 발굴. Claude Opus 4.6와 GPT-5.4는 합산 5.4%로 동률이나 **약점 시나리오가 서로 크게 다르다**.
- **🛠 agent-builder 시사점**: 프롬프트 인젝션 방어를 **단발 테스트로 통과했다고 안심하지 말 것**. 멀티턴 적응 공격 + 여러 공격 모델 풀링으로 회귀 세트를 구성하고, 모델별 약점 시나리오를 따로 추적하라.
- 링크: [arxiv.org/abs/2607.18063](https://arxiv.org/abs/2607.18063)

# 🛠 오픈소스 스포트라이트

## open-code-review (Alibaba) — 코드슬롭 시대의 리뷰 게이트
- **repo**: [github.com/alibaba/open-code-review](https://github.com/alibaba/open-code-review)
- **언어**: 주로 Java 계열(서버형 리뷰 파이프라인)
- **무엇을**: Alibaba 사내에서 2년간 수만 명 개발자에게 쓰이던 AI 코드리뷰 도구를 오픈소스화. **결정적(deterministic) 파이프라인 + LLM 에이전트**의 하이브리드 구조로, 파일 필터링·태스크 분할·위치 지정·동시성 같은 공정은 결정적으로, 의미 판단은 LLM으로 나눈다. NPE·스레드 안전성·XSS·SQL 인젝션 룰셋을 내장하고 OpenAI/Anthropic 호환 API를 지원, **라인 단위 코멘트**를 낸다.
- **에이전트 개발자 활용처**: TRIM이 말하는 "장황한 에이전트 패치"를 사람이 다 읽기 전에, 결정적 룰 + LLM 리뷰로 1차 게이트를 세울 수 있다. 에이전트가 연 PR에 자동 리뷰 봇으로 붙이기 좋다.

> ⚠️ **검증 메모**: 이번 세션은 GitHub API 접근이 이 블로그 리포로만 스코프돼 **외부 리포의 스타 수·최신 커밋을 API로 검증할 수 없었다**. 검색 기준 스타 수는 약 10.7k로 보이지만 확정 수치로 싣지 않는다. 도입 전 리포 활동성 지표는 직접 확인하길.

## 🟦 Claude Code — 팬아웃 상한과 예산을 명시로 잡기 (v2.1.217)
- 카테고리: Subagent / Context
- 출처: [code.claude.com/docs/en/changelog](https://code.claude.com/docs/en/changelog) (v2.1.217)
- 💡 **핵심**: 이제 동시 서브에이전트가 **기본 20개로 상한**이고, 서브에이전트는 **중첩 스폰을 기본 비활성**한다. 무한 팬아웃이 조용히 막히거나 잘릴 수 있다.
- 📝 **How-to**:
  ```bash
  # 대규모 병렬 리뷰/탐색은 상한을 명시적으로 올린다
  export CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS=40
  # 비용 상한은 이제 백그라운드 서브에이전트까지 중단시킨다
  claude --max-budget-usd 5.00
  ```
- 🎯 **언제**: 파일 수십 개를 서브에이전트로 병렬 분석/리뷰하는 파이프라인.
- ⚠️ **주의**: 중첩 서브에이전트에 의존하던 오케스트레이션은 동작이 바뀐다 — 계층 구조를 상위에서 평탄화하라.
- ⭐ **추천**: 상한과 예산을 CI 잡의 env로 고정해 "런어웨이 팬아웃"과 비용 폭주를 동시에 막기.

## 🟩 Codex — 원격 워크스페이스로 위험 작업 격리
- 카테고리: Workflow / Sandbox
- 출처: [Codex changelog](https://developers.openai.com/codex/changelog)
- 💡 **핵심**: 새 **DigitalOcean 플러그인**은 Codex가 Droplet을 프로비저닝하고 SSH를 붙여 **원격 워크스페이스**로 연결하게 한다. 파괴적/의존성-지저분한 작업을 로컬과 분리한다.
- 📝 **How-to**:
  ```text
  # 아이디어: 위험한 마이그레이션/빌드는 일회용 원격 워크스페이스에서
  1) 플러그인으로 Droplet 프로비저닝 → SSH 연결
  2) 에이전트 루프는 원격에서 실행(로컬 파일시스템 무접촉)
  3) 결과 diff만 회수해 로컬에서 리뷰 → 워크스페이스 폐기
  # 정확한 명령/플래그는 플러그인 문서를 따를 것
  ```
- 🎯 **언제**: `rm`/대량 설치/스키마 마이그레이션처럼 로컬을 오염시키고 싶지 않은 작업.
- ⚠️ **주의**: 원격 비용·시크릿 주입 경로를 관리하고, 다 쓴 Droplet은 반드시 폐기.
- ⭐ **추천**: "일회용 클라우드 샌드박스 → diff만 회수" 패턴을 위험 작업 기본값으로.

## 🟪 공통 — 커밋 전 '코드슬롭 다이어트' 게이트
- 카테고리: Review
- 출처: [arxiv.org/abs/2607.18161](https://arxiv.org/abs/2607.18161)
- 💡 **핵심**: 에이전트 diff는 탐색 잔여물로 부푼다(TRIM). 머지 전에 **불필요한 임시 변경·죽은 코드**를 걷어내는 단계를 넣어라.
- 📝 **How-to**:
  ```text
  # PR 체크리스트 (에이전트 생성 diff 전용)
  [ ] 최종 목표와 무관한 파일 변경이 섞여 있나?
  [ ] 디버그 로그·주석 처리된 실험 코드가 남았나?
  [ ] 같은 결과를 더 적은 변경으로 낼 수 있나? (사람 구현 대비 라인 수)
  → 남으면 에이전트에게 "동작 동일 + 최소 diff"로 재작성 요청
  ```
- 🎯 **언제**: 코딩 에이전트가 연 모든 PR.
- ⚠️ **주의**: 슬림화가 테스트 통과를 깨지 않는지 재검증(TRIM도 회귀 최소화가 핵심).
- ⭐ **추천**: 위 open-code-review 같은 리뷰 봇 + 이 체크리스트를 한 게이트로 묶기.
