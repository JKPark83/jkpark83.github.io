# 🔥 오늘의 핵심

오늘의 흐름은 모델이 아니라 **"에이전트에 능력을 어떻게 끼워 넣느냐"**다. Anthropic의 `anthropics/skills`(156.6k★)가 SKILL.md 표준+예제를 공개하면서 Skills가 자리를 잡았고, Codex도 같은 포맷을 문서로 지원한다. 그래서 오늘 할 일: 자주 반복하는 전문 작업을 프롬프트가 아니라 **SKILL.md 한 폴더**로 패키징해 Claude Code/Codex에 꽂아보라.

![오늘의 AI 변화 지도: Agent Skills를 중심으로 Claude Code, Codex, browser-use, MinerU를 연결](/images/ai-news-2026-06-30/change-map.svg)
*Skills 포맷을 축으로 오늘의 도구들을 연결한 지도. (직접 생성)*

# 🤖 모델 & 릴리즈 레이더

오늘은 큰 신규 가중치 공개 없음. 변화는 에이전트 런타임/CLI 쪽에 몰려 있다.

- **Codex `0.142.4` (stable, 6/29)**: 릴리스 노트상 **사용자 영향 변경 없음**("No user-facing changes"). 0.143 라인은 여전히 alpha(`0.143.0-alpha.29`, 6/28)로만 굴러간다. → 공유 CI/팀 환경은 stable `0.142.4`로 핀 고정, 인덱스 웹검색·멀티에이전트 위임 같은 신기능은 alpha에서만 시험. ([releases](https://github.com/openai/codex/releases))
- **Claude Code `v2.1.195` (6/26)**: 어제 이후 신규 버전 없음. 직전 변경은 하이픈 식별자(`code-reviewer`, `mcp__brave-search`) 후크 매처가 부분일치하던 버그를 **정확일치**로 고친 것. → 후크 매처를 쓰는 팀은 한 번 재점검할 값어치. ([changelog](https://code.claude.com/docs/en/changelog))
- **파운데이션 모델**: 6월 말 기준 프런티어급 신규 공개는 잠잠. 추정 가격·컨텍스트 수치는 신뢰 금지. 액션은 모델 교체보다 **에이전트 능력 패키징**에 있다.

# 📚 주목 논문

오늘도 arXiv 1차 검증 경로(`export.arxiv.org` API)와 Hugging Face Daily Papers가 네트워크 정책으로 막혀, ID·카테고리·결과 수치를 **1차 출처로 확인할 수 없었다**. 제로 페브리케이션 원칙에 따라 검증 안 된 논문은 싣지 않는다. 검증 경로가 복구되면 즉시 재개한다.

# 🛠 오픈소스 스포트라이트

테마는 "에이전트에 능력 주입". 모두 6/29~30에도 활발히 푸시 중인 현역 저장소다.

![오픈소스 능력 도구 스타 비교: anthropics/skills 156.6k, browser-use 101.4k, MinerU 72.3k](/images/ai-news-2026-06-30/repo-stars.svg)
*GitHub API 기준 스타 수. 막대 길이는 스타 수에 비례. (직접 생성)*

- **[anthropics/skills](https://github.com/anthropics/skills)** · Python · 약 156.6k★
  Agent Skills의 **표준(spec)+예제 모음**. 각 스킬은 `SKILL.md` 한 파일에 지침·메타데이터를 담고, 모델이 필요할 때 **동적으로** 로드한다. 문서 생성(docx/pdf/pptx/xlsx), 웹앱 테스트, MCP 서버 생성 같은 예제가 들어 있다. → 사내 브랜드 가이드·반복 워크플로를 재사용 가능한 단위로 묶고 싶을 때 출발점. 많은 스킬이 Apache 2.0.
- **[browser-use/browser-use](https://github.com/browser-use/browser-use)** · Python · 약 101.4k★
  "웹사이트를 에이전트가 쓸 수 있게" 만드는 브라우저 자동화 레이어. → 로그인·폼·검색처럼 **API가 없는 작업**을 에이전트 도구로 감쌀 때. Skills/도구 호출과 결합해 '웹 조작 능력'을 붙이는 표준 후보.
- **[opendatalab/MinerU](https://github.com/opendatalab/MinerU)** · Python · 약 72.3k★
  PDF·오피스 문서를 **LLM이 먹기 좋은 markdown/JSON**으로 변환. → RAG·문서 에이전트의 인입 파이프라인. 표·레이아웃이 깨지던 PDF를 정제해 컨텍스트 품질을 올린다.

비교 요약(능력 축):

- 능력 패키징 → `anthropics/skills`
- 웹 접근 → `browser-use`
- 문서 인입 → `MinerU`

# 🧑‍💻 Codex & Claude Code 실전 팁

### 🟦 Claude Code — anthropics/skills를 플러그인 마켓으로 등록하기 (Skill / Slash Command)
- 출처: [github.com/anthropics/skills](https://github.com/anthropics/skills)
- 💡 **핵심**: 스킬 저장소를 **플러그인 마켓플레이스**로 등록하면, 예제/문서 스킬을 골라 바로 설치할 수 있다.
- 📝 **how-to**:

```text
/plugin marketplace add anthropics/skills
# 이후: Browse and install plugins → anthropic-agent-skills 선택
# 또는 바로 설치
/plugin install document-skills@anthropic-agent-skills
/plugin install example-skills@anthropic-agent-skills
```

- 🎯 **언제**: docx/pdf/pptx/xlsx 생성, 웹앱 테스트 같은 반복 작업을 매번 프롬프트로 설명하기 싫을 때.
- ⚠️ **caveat**: README도 명시하듯 예제는 **데모/교육용**. 실제 동작은 환경에 따라 다르니, 크리티컬 작업 전 직접 검증.
- ⭐ **추천**: 자체 스킬을 만들기 전 패턴 학습용으로 가치 큼.

### 🟩 Codex — 같은 SKILL.md 포맷을 Codex에서도 (Skills / Config)
- 출처: [codex/docs/skills.md](https://github.com/openai/codex/blob/main/docs/skills.md)
- 💡 **핵심**: Codex도 Skills를 문서로 지원한다. 즉 `SKILL.md` 자산을 **두 도구에서 재사용**할 여지가 생긴다.
- 📝 **how-to**: 공유 환경은 stable로 고정하고, 스킬 폴더 구조(폴더당 `SKILL.md`)를 표준화해 둔다.

```bash
# 공유 CI/팀: 동작 고정용 stable
npm i -g @openai/codex@0.142.4 && codex --version
# 능력 단위는 폴더로: skills/<name>/SKILL.md
```

- 🎯 **언제**: Claude Code와 Codex를 오가며 같은 반복 작업을 시킬 때.
- ⚠️ **caveat**: 0.143은 alpha라 동작이 자주 바뀐다. 스킬 세부 동작은 도구별로 다를 수 있으니 양쪽에서 따로 검증.
- ⭐ **추천**: 멀티-CLI 팀이면 스킬을 단일 소스로 두는 정리가 이득.

### 🟪 공통 — SKILL.md "점진적 로딩"으로 컨텍스트 아끼기 (Context / Skill)
- 출처: [Anthropic Engineering — Equipping agents with Agent Skills](https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills)
- 💡 **핵심**: 스킬은 평소엔 `name`+`description`만 컨텍스트에 두고, 작업과 매칭될 때만 본문/스크립트를 펼친다. **능력은 늘리되 토큰은 덜 먹는다.**

![Agent Skill 동작 흐름: 메타데이터만 상주 → 필요 판단 → 리소스 사용 → 결과, 그리고 Claude Code/Codex 공통](/images/ai-news-2026-06-30/skills-flow.svg)
*SKILL.md의 점진적 로딩 흐름과 도구 간 공통성. (직접 생성)*

- 📝 **how-to**: `description`을 "언제 이 스킬을 써야 하는지"로 또렷하게 써라. 모델이 그걸 보고 로드를 결정한다. 무거운 절차·예시는 본문/별도 파일로 분리.
- 🎯 **언제**: 시스템 프롬프트가 비대해지고 있을 때, 능력은 많이 붙이되 상시 토큰은 줄이고 싶을 때.
- ⚠️ **caveat**: `description`이 모호하면 영영 로드 안 되거나 엉뚱하게 로드된다. 트리거 문구를 평가(eval)로 점검.
- ⭐ **추천**: 프롬프트 비대화에 시달리는 팀에 즉효.
