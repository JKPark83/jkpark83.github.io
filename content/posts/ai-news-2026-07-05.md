# 🧑‍💻 Codex & Claude Code 실전 팁

### 🟦 Claude Code — 슬래시 스킬 스태킹 (Slash Command / Workflow)
- 출처: [Claude Code changelog](https://code.claude.com/docs/en/changelog) · [skills 문서](https://code.claude.com/docs/en/skills)
- 💡 **핵심**: 2.1.199부터 **한 메시지 앞머리에 스킬을 여러 개 쌓아** 부를 수 있다. 첫 스킬 + **최대 5개**를 로드하고, 뒤에 붙인 텍스트는 **공유 `$ARGUMENTS`**로 각 스킬에 전달된다.
- 📝 **어떻게**:

```text
/code-review /fix-issue 123
```

  → `code-review`와 `fix-issue`가 함께 로드되고 `123`이 둘 다의 인자로 들어간다. 스킬에 `$ARGUMENTS`가 없으면 CC가 본문 끝에 `ARGUMENTS: <입력>`을 붙여준다.
- 🎯 **언제**: "리뷰 → 수정 → 테스트"처럼 **매번 같은 스킬 조합**을 굴릴 때. 매크로 하나로 파이프라인 시작.
- ⚠️ **주의**: 확장은 **인라인 스킬이 아닌 첫 토큰**에서 멈춘다. 포크된 서브에이전트나 `/loop` 같은 스킬이 오면 그 지점부터는 전부 **인자 텍스트**로 취급된다. 조합 순서에 유의.
- ⭐ **추천**: 반복 워크플로가 있으면 오늘 바로 조합 하나 만들어 두기.

### 🟩 Codex — 멀티에이전트 위임 튜닝 (Subagent / Workflow)
- 출처: [Codex Subagents 문서](https://developers.openai.com/codex/subagents)
- 💡 **핵심**: Codex는 **명시적으로 요청할 때만** 서브에이전트를 띄운다. 동시 스레드 수와 중첩 깊이는 `config.toml`의 `[agents]`로 제어한다.
- 📝 **어떻게**:

```toml
[agents]
max_threads = 6   # 동시 실행 스레드(기본 6)
max_depth = 1     # 중첩 깊이(기본 1: 자식은 스폰 가능, 더 깊은 재귀는 차단)
```

  트리거는 자연어로: "PR의 각 지적사항마다 에이전트 하나씩 띄우고, 전부 끝나면 항목별로 요약해줘." 스폰된 서브에이전트는 프로젝트 `AGENTS.md`를 그대로 읽으므로 팀 규칙이 자동 적용된다.
- 🎯 **언제**: 리뷰 포인트·파일·테스트처럼 **병렬로 쪼갤 수 있는** 작업을 한 번에 처리할 때.
- ⚠️ **주의**: `max_depth`는 특별한 이유가 없으면 **1로 유지**(무한 재귀 방지). 스레드를 늘리면 토큰·레이트리밋 소모도 함께 뛴다.
- ⭐ **추천**: 재귀 위임보다 **폭(max_threads)** 을 먼저 조절하는 편이 안전하고 예측 가능.

### 🟪 공통 — "안전 기본값(safe defaults)"으로 재점검 (Context / Review)
- 출처: [Claude Code 2.1.200 changelog](https://code.claude.com/docs/en/changelog)
- 💡 **핵심**: CC 2.1.200은 기본 권한 모드를 `default → **Manual**`로 바꾸고 **AskUserQuestion 자동 진행을 껐다**. 이는 에이전트 툴 전반의 방향 — **"의심스러우면 멈추고 물어봐라"**를 기본으로.
- 📝 **어떻게**: 팀/CI 설정에서 자동 진행에 의존하던 곳을 찾아 명시적으로 지정.

```jsonc
// .claude/settings.json — 새 기본값과 맞추기
{ "permissions": { "defaultMode": "manual" } }
// 자동 진행이 필요하면 /config 에서 idle timeout을 opt-in
```

- 🎯 **언제**: 백그라운드/무인 실행 파이프라인이 있는 팀. 조용히 흐르던 동작이 이제 멈춰 물어볼 수 있다.
- ⚠️ **주의**: `--permission-mode default`는 하위호환으로 아직 받지만, 문서·스크립트는 `manual`로 갱신 권장.
- ⭐ **추천**: 이 김에 에이전트의 **파괴적 액션(삭제·push·결제)** 게이팅을 한 번 감사하기.
