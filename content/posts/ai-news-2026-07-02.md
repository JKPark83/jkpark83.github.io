# 🔥 오늘의 핵심

오늘의 신호는 **"SOTA 코딩 에이전트 모델은 나왔는데, 아직 못 쓴다"**다. OpenAI가 7/1 추가 평가에서 **GPT-5.6 Sol**을 Terminal-Bench 2.1(에이전틱 CLI 작업) 최상위로 내세웠지만, 이 모델은 **한정 프리뷰**라 대량 트래픽에 붙일 수 없다. 그래서 오늘 개발자가 실제로 할 일은 두 가지: ① 지금 쓰는 **Codex는 0.142.5 stable로 올려** trace 로그 누출 이슈를 막고, ② 다음 모델을 믿기 전에 **Terminal-Bench/SWE-bench 하네스**를 미리 깔아 "리더보드 점수"를 "내 워크로드 델타"로 번역할 준비를 해두는 것이다.

# 🤖 모델 & 릴리즈 레이더

- **GPT-5.6 Sol, 7/1 추가 평가로 재조명 (OpenAI).** GPT-5.6 계열(**Sol**=플래그십 / **Terra**=밸런스 / **Luna**=경량, 그리고 고노력 모드 **Sol Ultra**)은 6/26 한정 프리뷰로 공개됐고, 7/1 벤치 업데이트에서 **Terminal-Bench 2.1** 최상위로 부각됐다. 2차 보도 기준 **Sol 88.8% / Sol Ultra 91.9%**인데, **OpenAI가 공식 수치·SWE-bench Verified를 공개하지 않았고** 공개 리더보드의 SWE-bench 상위는 여전히 Claude 계열이다 — 즉 "터미널/CLI 계열 작업"에 특화된 강점으로 읽어야 한다. **에이전트 빌더 입장:** 지금은 **약 20개 파트너 한정 프리뷰**(미 정부에 참여자 공유)라 프로덕션에 못 붙인다. GA 전까지는 가격표만 기억해 두자 — Sol **$5/$30**, Terra **$2.5/$15**, Luna **$1/$6**(M토큰 입력/출력). (출처: [edenai 정리](https://www.edenai.co/post/gpt-5-6-sol-benchmarks-pricing-api-access-guide), [vertu 가이드](https://vertu.com/guides/gpt-5-6-sol-benchmarks-release-status-2026))
- **Codex 0.142.5 stable 릴리스 (7/1).** stable이 0.142.4 → **0.142.5**로 올라갔다. 변경점은 **전체 Responses WebSocket 요청 페이로드가 trace 로그에 기록되던 문제 수정**(로깅/보안). 0.143 라인은 여전히 alpha만. **에이전트 빌더 입장:** trace를 켜고 돌리는 CI/샌드박스라면 페이로드(프롬프트·툴 인자) 누출을 막으려면 0.142.5로 올려라. (출처: [openai/codex releases](https://github.com/openai/codex/releases))
- **Claude Code는 조용한 날.** 최신은 여전히 v2.1.197(Sonnet 5 기본, 어제 다룸). 신규 버전 없음.

- 대신 오늘 주제와 직결되는 검증 가능한 자료: 아래 오픈소스 스포트라이트의 **Terminal-Bench / SWE-bench**가 GPT-5.6 Sol 헤드라인을 만든 바로 그 평가 하네스다. 논문 대신 "벤치가 실제로 뭘 재는지"를 코드로 읽어 보길 권한다. arXiv 경로가 복구되면 정식으로 다룬다.

# 🛠 오픈소스 스포트라이트

오늘 헤드라인(GPT-5.6 Sol의 Terminal-Bench 성적)을 **내 손으로 재현·검증**할 수 있게 해주는 두 하네스다. 모델을 못 쓰더라도, 하네스는 지금 깔 수 있다.

- **[laude-institute/terminal-bench](https://github.com/laude-institute/terminal-bench)** · Python · 약 2.4k★. LLM 에이전트를 **샌드박스 터미널**에 붙여 약 100개의 실제 작업(코드 컴파일, 모델 학습, 서버 셋업 등)으로 평가하는 벤치 프레임워크 + 실행 하네스. **활용:** GPT-5.6 Sol이 자랑한 바로 그 Terminal-Bench 2.1을 내 에이전트로 돌려, 리더보드 대신 **내 태스크에서의 성공률·툴콜 수**를 측정.
- **[SWE-bench/SWE-bench](https://github.com/SWE-bench/SWE-bench)** · Python · 약 5.3k★. "언어모델이 실제 GitHub 이슈를 고칠 수 있나"를 재는 표준 벤치. 실제 레포의 버그 패치 생성 능력을 평가한다. **활용:** 코딩 에이전트의 회귀를 잡는 고정 데이터셋으로 쓰고, 새 모델 승격 전후 패치 성공률 델타를 비교.

*두 하네스의 조합: Terminal-Bench(터미널/워크플로) + SWE-bench(이슈 해결). GPT-5.6은 앞쪽, Claude 계열은 뒤쪽에서 강하다는 게 오늘의 그림이다.*

# 🧑‍💻 Codex & Claude Code 실전 팁

### 🟩 Codex — Security / Debugging
- 출처: [openai/codex releases (0.142.5)](https://github.com/openai/codex/releases)
- 💡 **핵심:** trace 로깅을 켠 채 돌리면, 이전 stable(0.142.4 이하)에서는 **전체 WebSocket 요청 페이로드가 trace 로그에 통째로** 남았다. 프롬프트·툴 인자·컨텍스트가 그대로 파일에 떨어진다는 뜻.
- 📝 **방법:** stable을 0.142.5로 올리고, 공유 러너라면 버전을 핀 고정한다.

```bash
codex --version        # 0.142.5 인지 확인
npm i -g @openai/codex@0.142.5   # 또는 배포 방식에 맞게 핀 고정
```

- 🎯 **언제:** trace/verbose 로그를 남기는 CI·샌드박스, 로그를 아티팩트로 업로드하는 파이프라인.
- ⚠️ **주의:** 0.143은 아직 alpha다. 공유 CI는 0.142.5 stable로 유지.
- ⭐ **추천:** trace를 쓰는 팀이면 오늘 바로 업데이트.

### 🟦 Claude Code — Hook / Workflow
- 출처: [Claude Code Hooks 레퍼런스](https://code.claude.com/docs/en/hooks)
- 💡 **핵심:** 셀프호스트/원격 러너에서 세션이 끝나면 워크스페이스가 삭제된다. **`SessionEnd`(post-session) 훅**을 걸면 삭제 직전에 **커밋 안 된 작업·로그를 스냅샷**할 수 있다. 에이전트를 CI에서 돌릴 때 감사추적·복구용으로 필수급.
- 📝 **방법:** `.claude/settings.json`에 아래처럼 등록한다. 훅은 종료를 막을 수는 없고 정리 작업만 수행한다.

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          { "type": "command",
            "command": "~/.claude/snapshot.sh",
            "timeout": 120 }
        ]
      }
    ]
  }
}
```

- 🎯 **언제:** 원격/셀프호스트 러너에서 에이전트 세션을 자동으로 돌리고, 결과물·로그를 반드시 남겨야 할 때.
- ⚠️ **주의:** `SessionEnd`는 세션 종료를 **차단하지 못한다.** 스냅샷/업로드 같은 "끝난 뒤 정리"에만 쓰고, 타임아웃 안에 끝나게 짧게.
- ⭐ **추천:** 원격 세션을 자동화한다면 강력 추천.

### 🟪 공통 — Eval / Workflow
- 출처: [laude-institute/terminal-bench](https://github.com/laude-institute/terminal-bench) · [SWE-bench/SWE-bench](https://github.com/SWE-bench/SWE-bench)
- 💡 **핵심:** GPT-5.6 Sol의 "Terminal-Bench 최상위"처럼 **리더보드 점수는 내 워크로드 향상을 보장하지 않는다.** 특히 벤더가 공식 수치를 다 공개하지 않을 때는 더 그렇다.
- 📝 **방법:** ① Terminal-Bench/SWE-bench 하네스를 고정 → ② 내 실제 레포·이슈로 데이터셋 구성 → ③ 후보 모델을 **같은 하네스**에 태워 정확도·툴콜·토큰·비용 델타 측정 → ④ 델타가 양수일 때만 승격.

```text
새 모델 소문 ─→ Terminal-Bench/SWE-bench 하네스 ─→ 내 태스크 델타
                                                   └─ 양수면 승격 / 아니면 보류
```

- 🎯 **언제:** 모델이 격일로 쏟아지는 지금, "최신 추적"보다 "내 하네스로 빠른 검증"이 필요할 때.
- ⚠️ **주의:** 한정 프리뷰(예: GPT-5.6 Sol)는 아직 이 루프를 못 돌린다 — API가 열리기 전엔 파이프라인만 미리 구축.
- ⭐ **추천:** 모델 격변기일수록 ROI가 큰 패턴.
