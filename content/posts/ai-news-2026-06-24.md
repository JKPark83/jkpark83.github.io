# 📚 주목 논문

오늘은 논문 섹션을 **의도적으로 줄인다.** arXiv 1차 출처(`export.arxiv.org` API 및 `arxiv.org`)가 이 실행 환경의 송신 정책에서 차단되어, 규정대로 arXiv ID·카테고리·핵심 결과를 1차 검증할 수 없었다. ZERO 날조 원칙에 따라 **검증 못 한 논문은 싣지 않는다.** 내일 출처 접근이 회복되면 밀린 후보(예: 코딩 에이전트의 저장소 탐색 분리, 에이전트 플래닝 벤치마크)를 검증해 다룬다.

# 🛠 오픈소스 스포트라이트 — 폴백용 자체 RAG/에이전트 스택

프런티어 API가 막혀도 모델 레이어만 갈아끼우면 되도록, 워크플로는 내 인프라에 두자. 별점은 GitHub API로 오늘(2026-06-24) 조회한 값.

![Langflow·Dify·RAGFlow의 GitHub 별점 비교 막대 차트](/images/ai-news-2026-06-24/repo-stars.svg)
*세 빌더 모두 모델 공급자를 교체 가능한 레이어로 다룬다 — 그게 폴백의 핵심.*

- **[langflow-ai/langflow](https://github.com/langflow-ai/langflow)** · Python · 약 **150.0k ★**
  비주얼 노드 기반으로 RAG·에이전트 플로우를 조립·배포. 모델/임베딩을 노드 단위로 바꿔 끼울 수 있어, 공급자 교체 실험에 빠르다.
- **[langgenius/dify](https://github.com/langgenius/dify)** · TypeScript · 약 **146.3k ★**
  프로덕션 지향 에이전트 워크플로 플랫폼. 앱·API·관측을 한곳에서. 모델 라우팅을 설정으로 두면 폴백 전환이 코드 수정 없이 끝난다.
- **[infiniflow/ragflow](https://github.com/infiniflow/ragflow)** · Python · 약 **83.5k ★**
  깊은 문서 이해에 강한 오픈소스 RAG 엔진 + 에이전트. 검색 품질이 폴백 모델의 약점을 메워 준다.

**에이전트 개발자 활용:** 핵심 검색·오케스트레이션을 위 스택에 박아두고, 모델만 환경변수로 추상화하라. 프런티어가 막히는 날 평가셋만 다시 돌리면 된다.

# 🧑‍💻 Codex & Claude Code 실전 팁

### 🟦 Claude Code — `sandbox.credentials` (Security / Context)
- 출처: [Claude Code changelog v2.1.187](https://code.claude.com/docs/en/changelog)
- 💡 **핵심 아이디어:** 샌드박스에서 도는 명령이 자격증명 파일·비밀 환경변수를 읽지 못하게 차단. 셸 권한을 가진 에이전트의 유출 표면을 줄인다.
- 📝 **How-to** — `settings.json`:

```json
{
  "sandbox": {
    "credentials": true
  }
}
```

이 상태에서 에이전트가 `cat ~/.aws/credentials`나 `env | grep TOKEN`을 시도해도 비밀은 가려진 채 명령만 진행된다.

![sandbox.credentials가 에이전트 셸의 자격증명 접근을 차단하는 흐름](/images/ai-news-2026-06-24/sandbox-guard.svg)
*명령은 실행되되 비밀은 노출되지 않는다 — 프롬프트 인젝션 유출 표면을 좁힌다.*

- 🎯 **언제:** 셸 권한 에이전트를 CI·원격·미감독 러너에서 돌릴 때.
- ⚠️ **주의:** 정책이 아니라 안전망이다. 자격증명은 별도 시크릿 매니저로 주입하는 설계가 우선.
- ⭐ **추천:** 원격/배치 에이전트를 쓴다면 오늘 바로 켜라.

### 🟩 Codex — thread별 worktree + 환경 설정 스크립트 (Workflow)
- 출처: [Codex changelog](https://developers.openai.com/codex/changelog)
- 💡 **핵심 아이디어:** 새 thread마다 브랜치 선택 → git worktree 생성 → 환경 설정 스크립트 실행을 지원. 병렬 작업이 서로의 작업트리를 더럽히지 않는다.
- 📝 **How-to:** 새 작업을 별도 worktree로 격리하고, 의존성 설치 같은 준비를 셋업 스크립트로 자동화. 후속 작업에선 기존 PR을 그대로 업데이트할 수 있다.

```bash
# 예: 작업별 worktree로 격리 (개념 흐름)
git worktree add ../task-A feature/task-A
# Codex가 새 thread에서 위와 같은 격리 + setup script를 자동 수행
```

- 🎯 **언제:** 여러 작업을 동시에 굴리거나, 리뷰 중인 PR에 이어서 작업할 때.
- ⚠️ **주의:** worktree가 쌓이면 디스크·혼선을 부른다 — 끝난 건 정리하는 습관.
- ⭐ **추천:** 멀티태스킹 에이전트 운용 시 격리 단위를 worktree로 통일하면 충돌이 확 준다.

### 🟪 공통 — 모델 종속을 평가로 끊어라 (Eval / Governance)
- 💡 **핵심 아이디어:** 오늘의 핵심 리스크(프런티어 접근 단절)는 "폴백 모델로 같은 품질이 나오는가?"를 평소에 측정해두면 사고가 아니라 스위치가 된다.
- 📝 **How-to:** 평가셋을 모델 비종속으로 고정 → 주 모델과 폴백 모델 양쪽에 같은 셋을 돌려 점수·도구호출수·토큰을 기록 → 회귀가 임계치 안이면 폴백을 "프로덕션 가능"으로 표시.
- 🎯 **언제:** 단일 공급자에 매출/서비스가 묶여 있을 때.
- ⚠️ **주의:** 정확도만 보지 말 것 — 도구 호출 횟수·지연·비용까지 같은 셋으로 비교해야 진짜 동등성이다.
- ⭐ **추천:** "폴백을 켤 수 있다"와 "폴백을 검증해 뒀다"는 다르다. 오늘 후자로 만들어 두자.
