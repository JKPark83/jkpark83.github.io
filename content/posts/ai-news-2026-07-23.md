# 🔥 오늘의 핵심

오늘의 키워드는 **라우팅**이다. 두 편의 arXiv(둘 다 7/21)가 각각 "에이전트 **비용**을 라우팅으로 깎기"(CodeRescue)와 "에이전트 **기억**을 라우팅으로 살리기"(Supra Cognitive Modes)를 다룬다. CodeRescue는 코딩 에이전트가 실패했을 때 **바로 비싼 모델로 승격하지 말고** 값싼 재시도부터 태우는 라우터로 **항상-승격 대비 비용을 35%**까지 낮췄다. 오늘 할 일: 에이전트 루프의 "실패 → 복구" 경로에 **비용 라우터**를 하나 끼워 넣고, 메모리는 질의 유형별로 나눠서 조회하라.

# 📚 주목 논문

## 1) CodeRescue — 실패한 코딩 에이전트를 '예산 보정 라우팅'으로 구제
- **저자**: Qijia He, Jiayi Cheng, Chenqian Le, Rui Wang 외 (arXiv `2607.19338`, cs.AI, 2026-07-21 제출)
- **핵심**: 코딩 에이전트가 실행에 실패하면 보통 **더 강한(비싼) 모델로 승격**한다. 이 논문은 승격 대신 **재시도-저가 vs 승격**을 라우터가 판단하게 해 비용을 아낀다.
- **방법(요약)**: 실행 피드백과 **conformal risk control**로 예산을 보정한 라우터를 학습. "회복 가능해 보이는 실패"는 값싼 재시도로, "정말 막힌 것"만 승격으로 보낸다.
- **결과(검증됨)**: **항상-승격(always-escalate) 베이스라인보다 높은 해결률**을 내면서 **평균 복구 비용은 그 35%**만 소모.
- **🛠 agent-builder 시사점**: 코딩 에이전트 루프의 "실패 → 복구" 경로를 이진(재시도/승격)으로 나누고, 승격은 **예산·리스크 게이트를 통과할 때만** 하라. 그것만으로 실비용이 크게 준다.
- **한계**: 코딩 에이전트 시나리오 중심 평가라, 라우터 학습·보정을 팀 스택에 옮길 때의 이식 비용은 별도 측정 필요.
- 링크: [arxiv.org/abs/2607.19338](https://arxiv.org/abs/2607.19338)

![CodeRescue 라우팅 다이어그램. 코딩 에이전트가 테스트 실패하면 복구 라우터가 저가 모델 재시도와 강한 모델 승격을 판단하고, 항상-승격보다 높은 해결률을 그 35% 비용으로 달성한다](/images/ai-news-2026-07-23/coderescue-routing.svg)
*무작정 승격이 아니라 '실패의 종류'를 보고 라우팅하는 게 핵심. (자체 제작 SVG, 값 출처 arXiv 2607.19338)*

## 2) Supra Cognitive Modes — 에이전트 메모리를 질의 유형별로 라우팅
- **저자**: Joshua Tobkin, David Yang (arXiv `2607.19096`, cs.AI, 2026-07-21 제출)
- **핵심**: 모든 메모리 질의를 한 파이프라인으로 처리하지 않고, **직접 사실 조회 / 관계-체인 추론 / 광범위 종합**의 세 모드로 **라우팅**한다.
- **방법(요약)**: 의미 분류 + 런타임 게이트로 질의를 판별해 각 모드에 맞는 검색·종합 페이로드를 태운다.
- **결과(검증됨)**: LoCoMo factoid **84.87%**, MemoryAgentBench **61.49%**, LongMemEval **86.00%**.
- **🛠 agent-builder 시사점**: RAG/메모리를 단일 벡터검색으로만 돌리고 있다면, "이 질문은 사실 조회인가, 관계 추론인가, 종합인가"를 먼저 분기해 각기 다른 검색 전략을 붙여보라.
- **한계**: 저자 스스로 **통계적 유의성과 인과적 라우팅 효과는 미확립**이라 밝힘 — 자기 데이터로 A/B 검증 후 도입할 것.
- 링크: [arxiv.org/abs/2607.19096](https://arxiv.org/abs/2607.19096)

![에이전트 메모리 벤치마크 막대 차트. LoCoMo 84.87, MemoryAgentBench 61.49, LongMemEval 86.00 퍼센트](/images/ai-news-2026-07-23/agent-memory-bench.svg)
*세 벤치의 정확도. 라우팅으로 장기 메모리 태스크를 끌어올린다. (자체 제작 SVG, 값 출처 arXiv 2607.19096)*

# 🛠 오픈소스 스포트라이트

## Rowboat — "기억하는" AI 코워커
- **repo**: [github.com/rowboatlabs/rowboat](https://github.com/rowboatlabs/rowboat)
- **언어**: 주로 TypeScript/Python 계열(로컬 실행형 코워커)
- **무엇을**: 이메일·회의 노트에 연결해 **장기 지식 그래프**를 쌓고, 그 맥락으로 일을 돕는 오픈소스 AI 코워커. 세션을 넘어 **지속 메모리**를 유지하고, **MCP**로 검색·DB·CRM·사내 툴에 연결한다. 새 메일 같은 이벤트나 매일 정해진 시간에 **백그라운드 에이전트**를 돌릴 수 있다(YC 지원).
- **에이전트 개발자 활용처**: 위 Supra Cognitive Modes가 말한 "메모리가 매번 콜드 스타트하지 않게" 하는 실전 레퍼런스. 이벤트·스케줄 트리거 + MCP 툴 연결 패턴을 그대로 참고하기 좋다.

## Strix — PoC 검증형 펜테스트 에이전트
- **repo**: [github.com/usestrix/strix](https://github.com/usestrix/strix)
- **언어**: Python (Apache-2.0)
- **무엇을**: 정적 스캐너가 아니라 **계획·조사·검증을 끝까지 수행**하는 멀티-에이전트 보안 도구. OWASP Top 10 등에서 취약점을 찾아 **실제 PoC 익스플로잇으로 검증**해 false positive를 걷어낸다. CI/CD 통합과 skills 문서 제공.
- **에이전트 개발자 활용처**: "에이전트가 만든 코드/파이프라인"을 **검증형 에이전트로 되받아치는** 방어 루프에 붙일 수 있다. 자동화된 PR에 보안 검증 게이트로 두기.

> ⚠️ **검증 메모**: 이번 세션의 GitHub API는 이 블로그 리포로만 스코프돼 **외부 리포의 스타 수·최신 커밋을 API로 확정할 수 없었다**. 웹 기준 규모는 Rowboat ~16k, Strix ~41k로 보이지만 **확정 수치로 싣지 않는다**. 도입 전 활동성·라이선스는 직접 확인하길.

## 🟩 Codex — 롱러닝 작업의 승인 게이트를 모바일로 (Codex Remote GA)
- 카테고리: Workflow
- 출처: [developers.openai.com/codex/changelog](https://developers.openai.com/codex/changelog)
- 💡 **핵심**: Codex Remote가 GA. 연결된 **Mac/Windows 호스트**의 에이전트 작업을 폰에서 시작·재개·**승인**한다. QR 1:1 페어링으로 인증.
- 📝 **How-to**:
  ```text
  # 자리 비운 사이에도 위험 액션만 폰으로 승인
  1) 데스크톱 호스트에 Codex 연결 → 모바일 앱과 QR 1:1 페어링
  2) 긴 리팩터/마이그레이션은 호스트에서 백그라운드 실행
  3) 위험 액션(파일 삭제·배포·시크릿 접근)만 모바일 알림으로 승인
  4) 자동 승인 정책은 좁게: 파괴적 작업은 항상 수동 승인 유지
  ```
- 🎯 **언제**: 수십 분~시간 단위로 도는 에이전트 작업을 두고 자리를 떠야 할 때.
- ⚠️ **주의**: 6/8 이후 안 쓴 연결은 재페어링 필요. 모바일 승인은 편해서 남발하기 쉬우니 **파괴적 작업은 자동승인에서 제외**.
- ⭐ **추천**: "호스트에서 실행 + 위험 액션만 폰 승인"을 롱러닝 작업 기본값으로.

## 🟪 공통 — 실패 복구는 '재시도-저가부터', 승격은 게이트를 통과할 때만
- 카테고리: Workflow / Cost
- 출처: [arxiv.org/abs/2607.19338](https://arxiv.org/abs/2607.19338)
- 💡 **핵심**: 에이전트가 실패하면 곧바로 최강 모델로 올리지 말 것(CodeRescue). **값싼 재시도 → 안 되면 승격** 순으로만 태워도 실비용이 크게 준다.
- 📝 **How-to**:
  ```text
  # 실패 복구 라우팅(의사코드)
  on_failure(task):
      if looks_recoverable(exec_feedback) and budget_ok():
          retry_with_cheap_model(task)      # 1차: 저가 재시도
      elif risk_gate_pass(task):
          escalate_to_strong_model(task)    # 2차: 게이트 통과 시에만 승격
      else:
          stop_and_ask_human(task)          # 그 외: 사람에게
  ```
- 🎯 **언제**: 코딩/툴콜 에이전트 루프에서 실패·재시도가 잦은 구간.
- ⚠️ **주의**: "회복 가능" 판정 로직이 없으면 무한 저가 재시도로 되레 비싸질 수 있다 — 재시도 상한·예산 게이트를 반드시 둘 것.
- ⭐ **추천**: 실패 경로를 **이진 라우팅(재시도/승격) + 예산 게이트**로 명시화. 항상-승격 대비 비용 1/3이 목표.
