# 📚 주목 논문

오늘은 arXiv(`export.arxiv.org`)가 열려 ID·저자·수치를 1차 확정했다. 둘 다 "에이전트를 안 고치고 옆에 붙이는" 플러그형 접근이라 실전 적용이 빠르다.

### 1) 능동 메모리 에이전트 — "언제 상기시킬지"를 결정
- 저자: Yifan Wu, Lizhu Zhang, Yuhang Zhou 외 · **arXiv 2607.08716 (cs.AI)**
- 핵심 기여: 긴 궤적에서 결정에 필요한 상태가 컨텍스트 밖으로 밀리는 **"행동 상태 붕괴(behavioral state decay)"**를, **수동 검색이 아닌 능동 개입**으로 막는다.
- 방법: 액션 에이전트는 **손대지 않고**, 별도 **메모리 에이전트**가 최근 궤적으로 구조화된 메모리 뱅크를 갱신하고 **'리마인더 주입 vs 침묵'을 매 순간 결정**. 기존 하네스에 plug-and-play.
- 결과(검증): pass@1이 **Terminal-Bench 2.0 +8.3pp, τ²-Bench +6.8pp**. 약한/강한 액션 에이전트 모두 상승. 어블레이션에서 **선택적 주입 > 항상 주입 > 수동 노출 > 일반 검색**.
- 🛠 **빌더 시사점**: 컨텍스트 창을 키우는 대신, 궤적을 감시하다 **필요할 때만 목표·제약을 다시 꽂는** 사이드카를 붙여라.
- ⚠️ 한계: 오픈웨이트 메모리 정책(Qwen3.5-27B, SFT+GRPO)은 아직 Terminal-Bench로 **부분 전이**에 그침.
- 링크: [arXiv:2607.08716](https://arxiv.org/abs/2607.08716)

![능동 메모리 에이전트 방법 다이어그램: 액션 에이전트 옆에서 메모리 에이전트가 선택적으로 리마인더 주입](/images/ai-news-2026-07-11/memory-agent.svg)
*메모리를 '패시브 검색'이 아니라 '능동 개입'으로 본다 — 뱅크 갱신 후 주입할지 말지를 스스로 판단.*

### 2) WebSwarm — 재귀 위임 멀티에이전트 웹 탐색
- 저자: Xiaoshuai Song, Liancheng Zhang 외 · **arXiv 2607.08662 (cs.CL)**
- 핵심: 단일 ReAct 에이전트의 **긴 단일 궤적·제한된 컨텍스트** 한계를, **작업 분해 + 재귀 확장 + 협업**을 추론 중 동시에 구성해 넘는다. 각 노드가 직접 풀거나 **자식 노드로 더 위임**하고 근거를 위로 올려 부모가 확장·수정·집계.
- 결과(검증): BrowseComp-Plus·WideSearch·DeepWideSearch·GISA에서 단일/멀티 에이전트 베이스라인을 **일관되게 상회**.
- 🛠 **빌더 시사점**: "깊게(depth) vs 넓게(coverage)"를 한 에이전트에 욱여넣지 말고 **재귀 위임 트리**로 분리 — 형제 노드 간 과정 경험 재사용이 핵심.
- 링크: [arXiv:2607.08662](https://arxiv.org/abs/2607.08662)

# 🛠 오픈소스 스포트라이트

솔직히 밝힌다 — **오늘은 GitHub(API·HTML 모두)가 네트워크 정책으로 차단**돼 star 수치를 1차 확인할 수 없었다. 제로 팩트 원칙상 검증 불가한 별점으로 레포 스포트라이트를 채우지 않고, 대신 **오늘 바로 재현 가능한 검증된 아티팩트**를 가리킨다.

- **오늘의 두 논문이 곧 '오픈 레시피'**다. 능동 메모리 에이전트는 **기존 하네스에 붙는 사이드카**(별도 코드 에이전트 불필요), WebSwarm은 **재귀 위임 스캐폴딩** 패턴이다. 둘 다 프레임워크 종속이 없어, 지금 쓰는 오케스트레이터(LangGraph·자체 루프 등)에 개념만 이식하면 된다.
- 평가 기반은 이미 공개된 **Terminal-Bench 2.0 / τ²-Bench**. 메모리 사이드카 적용 전후를 이 벤치로 A/B 하면 위 +8.3pp가 내 스택에서도 재현되는지 바로 확인된다.

### 🟦 Claude Code — 커밋·PR 마찰 줄이고 CLAUDE.md 다이어트 (category: Workflow)
- 출처: [Claude Code 체인지로그](https://code.claude.com/docs/en/changelog) (2.1.206, 7/9)
- 💡 **핵심**: `/commit-push-pr`이 **설정된 push remote로의 `git push`를 자동 허용**해 커밋→푸시→PR 한 흐름의 승인 클릭이 준다. 덤으로 `/doctor`가 **비대해진 체크인 `CLAUDE.md`를 줄이라고 제안**한다.
- 📝 **how-to**:

```
/doctor        # 체크인된 CLAUDE.md가 너무 크면 트리밍 제안이 뜬다
/commit-push-pr  # 이제 push remote로의 push는 매번 승인 없이 진행
```

- 🎯 **언제**: 하루에도 여러 번 커밋·PR을 돌리는 반복 루프, 그리고 컨텍스트를 잡아먹는 거대 `CLAUDE.md`를 쓰는 리포.
- ⚠️ **주의**: 자동 push는 **설정된 push remote 한정**이다. 보호 브랜치·리뷰 게이트가 있는 팀은 remote 지정을 먼저 점검.
- ⭐ **추천**: 승인.  CLAUDE.md 다이어트는 컨텍스트 예산에 직결되니 정기적으로 돌려라.

- 출처: [Codex 체인지로그](https://developers.openai.com/codex/changelog) (rust-v0.143.0, 7/7)
- 💡 **핵심**: **원격 플러그인이 기본 활성화**되고 카탈로그 정보가 풍부해졌다 — 팀이 공유 플러그인/MCP 세트를 배포하면 멤버가 **수동 설치 없이** 받는다.
- 📝 **how-to**: 팀 표준 툴(사내 검색·이슈 트래커 MCP 등)을 카탈로그로 묶어 배포 → 신규 멤버는 최신 Codex만 깔면 동일한 툴 표면을 그대로 얻는다. macOS/Windows는 **프록시 인식 인증**도 함께 붙었다.
- 🎯 **언제**: 여러 명이 같은 에이전트 툴 세트를 공유해야 하는 팀 온보딩.
- ⚠️ **주의**: 원격 플러그인은 곧 **공급망 표면**이다. 카탈로그 출처·권한을 반드시 검토하고, 신뢰된 소스만 등록.
- ⭐ **추천**: 팀이면 승인, 개인 단독이면 굳이.

### 🟪 공통 — '메모리 체크포인트'를 에이전트 루프에 심어라 (category: Context / Memory)
- 출처: [arXiv:2607.08716](https://arxiv.org/abs/2607.08716)
- 💡 **핵심**: 오늘 논문의 실전 번역 — 컨텍스트 창을 무작정 키우지 말고, **"지금 결정에 필요한데 궤적에 묻힌 상태가 있나?"**를 주기적으로 묻고 **필요할 때만** 목표·제약·미해결 서브골을 다시 꽂아라.
- 📝 **how-to**: N스텝마다(또는 서브태스크 경계에서) 얇은 체크포인트 스텝을 넣는다.

```
# 매 서브태스크 경계에서 실행하는 메모리 체크포인트(의사코드)
buried = memory_agent.scan(recent_trajectory)   # 묻힌 요구사항/제약/이전 시도
if buried.decision_relevant:                     # 항상이 아니라 '선택적'으로
    inject_reminder(buried.top_k)                # 목표·제약만 간결히 재주입
```

- 🎯 **언제**: 20+ 스텝짜리 긴-호라이즌 코딩/리서치 에이전트에서 초반 지시를 자꾸 잊을 때.
- ⚠️ **주의**: **항상 주입은 역효과**(어블레이션에서 선택적 주입에 밀림) — 노이즈가 되니 게이팅이 핵심.
- ⭐ **추천**: 긴 잡을 돌리는 팀이면 강력 추천. 전체 히스토리 덤프보다 싸고 정확하다.
