# 🔥 오늘의 핵심

오늘 새 프런티어 모델 소식은 없다(Sonnet 5·GPT-5.6 Sol 이후 소강). 대신 세 흐름이 한 점 **"에이전트 기억(memory)"**으로 모인다. **Claude Code 2.1.198**은 백그라운드 에이전트가 멈추거나 끝날 때 쏘는 **Notification 훅**을, **Codex**는 스레드를 넘나드는 **Memories**를 정식 기능으로 문서화했다. 기억을 담는 인프라는 **mem0·letta**가 맡는다. 그래서 오늘 할 일은 명확하다 — 기억을 붙이되, **"주입한 기억이 실제로 성능을 올렸는가"를 SkillsBench식 A/B로 검증**한 뒤에만 프로덕션에 올려라.

# 📚 주목 논문

> ⚠️ 오늘은 arXiv 원문 접근이 막혀 **ID·제목은 검색 인덱스로 확인**했지만 내부 수치는 직접 검증하지 못했다. 그래서 2편으로 줄이고, **벤치 점수는 인용하지 않는다**(원문에서 확인 권장).

**1) SkillsBench: Benchmarking How Well Agent Skills Work Across Diverse Tasks**
- arXiv: **2602.12670** (cs.AI 계열)
- 핵심 기여: 11개 도메인 **86개 태스크**를 큐레이션된 Skill + **결정적 검증기**와 짝지어, 각 태스크를 **세 조건(스킬 없음 / 큐레이션 스킬 / 자기생성 스킬)**으로 평가하는 벤치.
- 🛠 agent-builder 시사점: "스킬/메모리를 붙였으니 좋아졌겠지"를 **가정하지 말고 3-조건 A/B로 측정**하라. 자기생성 스킬이 큐레이션만큼 통하는지가 실전 관건.
- 한계: 태스크·검증기 커버리지가 곧 결과를 좌우 → 내 워크로드로 재현 필요.
- 링크: https://arxiv.org/abs/2602.12670

**2) SWE-Skills-Bench: Do Agent Skills Actually Help in Real-World SWE?**
- arXiv: **2603.15401**
- 핵심 기여: 실제 소프트웨어 엔지니어링 태스크에서 **에이전트 스킬의 한계효용(marginal utility)을 분리 측정**.
- 🛠 시사점: 코딩 에이전트에 스킬 팩을 얹기 전, **"이 스킬이 없을 때 대비 얼마나 이득인가"**를 별도 축으로 재라.
- 한계: SWE 도메인 특화 → 비코딩 에이전트로 일반화는 조심.
- 링크: https://arxiv.org/abs/2603.15401

# 🧑‍💻 Codex & Claude Code 실전 팁

## 🟩 Codex — Context/Memory
- 소스: [developers.openai.com/codex/memories](https://developers.openai.com/codex/memories)
- 💡 핵심: 매 스레드마다 프로젝트 규칙을 다시 설명하지 말고 **Memories를 켜서** 선호·컨벤션·함정을 스레드 간에 이어붙여라.
- 📝 방법:

```toml
# ~/.codex/config.toml
[features]
memories = true
# 저장 위치: ~/.codex/memories/  (요약·durable 항목·최근 입력·근거)
```

- 🎯 언제: 같은 레포/스택을 반복 작업할 때. **AGENTS.md**(레포에 동행하는 지침)와 조합하면 "지침=규칙, memories=학습된 로컬 컨텍스트"로 역할 분리.
- ⚠️ 주의: 팀 공유 레포에 개인 선호가 새지 않게 저장 범위를 확인. 민감정보는 기억시키지 말 것.
- ⭐ 추천: 반복 프로젝트라면 강력 추천. 일회성 작업엔 불필요.

## 🟦 Claude Code — Hook
- 소스: [CHANGELOG 2.1.198](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) · [Hooks 문서](https://code.claude.com/docs/en/hooks)
- 💡 핵심: 백그라운드 에이전트가 **입력 대기(`agent_needs_input`)**하거나 **완료(`agent_completed`)**될 때 Notification 훅으로 외부 알림(슬랙/데스크톱)을 쏴라. 화면을 지키지 않아도 된다.
- 📝 방법:

```json
{
  "hooks": {
    "Notification": [
      {
        "matcher": "agent_needs_input|agent_completed",
        "hooks": [
          { "type": "command", "command": "~/.claude/notify.sh" }
        ]
      }
    ]
  }
}
```

- 🎯 언제: 백그라운드 다중 에이전트를 "던져두고" 다른 일 할 때. 2.1.198부터 완료 시 자동 commit·push·draft PR도 되므로 알림만 받으면 리뷰로 직행.
- ⚠️ 주의: Notification 훅은 **비차단(non-blocking)** — 실패해도 에이전트를 막지 못하니 게이트 용도로 쓰지 말 것. `notify.sh`는 짧게.
- ⭐ 추천: 백그라운드 에이전트 쓰는 사람은 오늘 바로 설정 권장.

## 🟪 공통 — Eval
- 소스: [SkillsBench (arXiv 2602.12670)](https://arxiv.org/abs/2602.12670)
- 💡 핵심: 메모리/스킬을 "켜기"보다 **"켠 게 실제로 이득인지"** 검증이 먼저다.
- 📝 방법: 내 태스크 셋을 **3-조건**으로 돌려 Δ를 본다 — ① 메모리 없음(베이스라인) ② 큐레이션 기억 주입 ③ 자기생성 기억 주입. **정확도·툴콜 수·토큰**을 함께 기록.
- 🎯 언제: mem0/letta나 Codex Memories를 프로덕션에 올리기 직전.
- ⚠️ 주의: 무한 저장은 검색 노이즈·토큰 폭증을 부른다. Δ가 (+)일 때만 승격.
- ⭐ 추천: 메모리를 붙였다면 필수 절차.
