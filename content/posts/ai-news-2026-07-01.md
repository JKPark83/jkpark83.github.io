# 🔥 오늘의 핵심

# 🛠 오픈소스 스포트라이트

Sonnet 5처럼 "모델만 갈아끼우는" 시대일수록, **게이트웨이 + 관측/평가**를 깔아두면 교체 비용·리스크가 0에 수렴한다. 오늘 묶음은 그 스택이다.

- **[BerriAI/litellm](https://github.com/BerriAI/litellm)** · Python · 약 52.2k★. 100+ LLM API를 OpenAI 포맷으로 통일해 호출하는 SDK/프록시(게이트웨이). **활용:** 코드 한 줄 안 바꾸고 라우팅 설정만으로 Sonnet 5로 전환, 모델별 비용·레이트리밋·폴백을 한곳에서 제어.
- **[langfuse/langfuse](https://github.com/langfuse/langfuse)** · TypeScript · 약 30.2k★. 오픈소스 LLM 엔지니어링 플랫폼(트레이스·평가·프롬프트 관리). **활용:** 모델 교체 전후 동일 데이터셋으로 eval을 돌려 품질 회귀를 잡고, 에이전트 멀티스텝 트레이스를 시각화.
- **[comet-ml/opik](https://github.com/comet-ml/opik)** · Python · 약 20.2k★. 에이전트/RAG 워크플로 디버깅·평가·모니터링 툴. **활용:** 툴콜 단위로 실패 지점을 추적하고 자동 eval 룰로 회귀를 감시.

# 🧑‍💻 Codex & Claude Code 실전 팁

### 🟦 Claude Code — Context / Workflow
- 출처: [Claude Code v2.1.197·v2.1.196 changelog](https://github.com/anthropics/claude-code/releases)
- 💡 **핵심:** Sonnet 5가 기본이 되면서 **1M 컨텍스트**가 손에 들어왔다. 단, 팀이라면 모델 드리프트를 막아야 한다.
- 📝 **방법:** 업데이트 후 모델 확인·전환은 `/model`, 컨텍스트 점유는 `/context`로 본다. 조직은 관리자 콘솔의 **org default model**로 기본값을 고정하면 `/model`에 "Org default"로 표시된다.

```text
# 업데이트 후
$ claude --version      # 2.1.197 이상인지
/model                  # 기본이 Sonnet 5인지 / Org default 표시 확인
/context                # 1M을 다 쓰지 말 것 — 필요한 파일만 주입
```

- 🎯 **언제:** 긴 코드베이스를 한 번에 읽혀야 하는 에이전트 작업, 또는 팀 비용 통제가 필요할 때.
- ⚠️ **주의:** 1M이 있다고 전부 넣으면 비용·지연·노이즈가 늘어 정확도가 오히려 떨어진다. 컨텍스트는 "넣을 수 있는 양"이 아니라 "필요한 양"으로.
- ⭐ **추천:** 강력 추천. 업데이트→`/model` 확인은 오늘 바로.

### 🟩 Codex — Hook / Security
- 출처: [openai/codex docs/config.md](https://github.com/openai/codex/blob/main/docs/config.md)
- 💡 **핵심:** 공유/CI 환경에서 신뢰할 수 없는 레포의 lifecycle hook이 멋대로 실행되는 걸 막을 수 있다.
- 📝 **방법:** 관리자가 `requirements.toml` 최상단에 아래를 두면, 관리형(승인된) hook만 실행된다.

```toml
# requirements.toml (관리자/조직 레벨)
allow_managed_hooks_only = true
```

- 🎯 **언제:** 외부 레포를 자동으로 끌어와 Codex로 돌리는 CI·샌드박스.
- ⚠️ **주의:** 끄면 레포에 포함된 hook이 임의 명령을 실행할 수 있다 — 프롬프트 인젝션/공급망 리스크.
- ⭐ **추천:** 공유 러너라면 사실상 필수.

### 🟪 공통 — Eval / Workflow
- 출처: [BerriAI/litellm](https://github.com/BerriAI/litellm) · [langfuse/langfuse](https://github.com/langfuse/langfuse)
- 💡 **핵심:** 모델 교체(예: → Sonnet 5)는 "코드 수정"이 아니라 "설정 + 회귀 테스트"여야 한다.
- 📝 **방법:** ① LiteLLM 게이트웨이로 모델을 설정값화 → ② 교체 전후 같은 eval 데이터셋을 Langfuse로 실행 → ③ 비용/정확도 델타가 양수일 때만 승격.

```text
앱 ─→ LiteLLM(gateway) ─→ [claude-sonnet-5 | gpt-* | ...]
          └─ trace ─→ Langfuse(eval/관측) ─→ 비용·품질 델타 확인 후 승격
```

- 🎯 **언제:** 새 모델이 뜰 때마다(요즘은 거의 격일) 빠르게 검증하고 싶을 때.
- ⚠️ **주의:** "벤치마크가 올랐다"는 내 워크로드에서의 향상을 보장하지 않는다. 반드시 자체 데이터셋으로 측정.
- ⭐ **추천:** 모델 격변기일수록 ROI가 큰 패턴.
