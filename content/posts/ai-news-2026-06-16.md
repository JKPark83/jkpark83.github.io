# 🔥 오늘의 핵심 트렌드

오늘은 "에이전트를 어떻게 더 잘 학습·관리할 것인가"라는 질문에 세 편의 논문이 동시에 답했다. **HarnessX**는 실행 추적으로 하네스 자체를 자동 진화시켜 SWE-bench 등 5개 벤치마크에서 평균 +14.5%를 기록했고, **APPO**는 도구 호출 경계에 머물던 RL 크레딧 할당을 시퀀스 전체로 확장해 13개 벤치마크에서 근 +4p를 달성했다. 멀티에이전트 메모리·오케스트레이션 논문도 동시에 쏟아졌으며, Google DeepMind의 Shane Legg 등 14인이 AGI 이후 ASI 전환 경로를 체계화한 보고서를 공개해 장기 전략 논의에 불을 붙였다.

## [1] HarnessX: A Composable, Adaptive, and Evolvable Agent Harness Foundry

- **저자/소속** 다수 (14인+), **arXiv** 2606.14249, **카테고리** cs.AI / cs.LG
- **핵심 기여**: 실행 추적 기반 다중 에이전트 진화 엔진(AEGIS)으로 프롬프트·도구·메모리·제어 흐름을 자동으로 개선하는 자가 진화 에이전트 하네스 프레임워크. "에이전트 발전이 모델 스케일링만으로 올 필요는 없다"는 것을 수치로 입증.
- **방법론 요약**: 하네스 컴포넌트를 타입 정의된 대수 치환(substitution algebra)으로 조립하고, AEGIS가 실행 궤적을 상징적 적응과 RL의 이중 피드백 루프로 처리해 하네스 업데이트와 학습 신호를 동시에 생성한다. 각 컴포넌트는 타입 프리미티브로 교환 가능하며, 실행 후 자동으로 파라미터를 조정한다. 기저 성능이 낮은 과제일수록 개선 폭이 더 컸다.
- **주요 결과**: ALFWorld·GAIA·WebShop·tau³-Bench·SWE-bench Verified 5개 벤치마크 평균 **+14.5%**, 단일 벤치마크 최대 **+44.0%**
- **🛠 실무 시사점**:

  | 적용 분야 | 기대 효과 |
  |---|---|
  | 사내 코드 리뷰 에이전트 | 누적 실행 로그로 리뷰 정확도 자동 향상 |
  | QA 자동화 파이프라인 | 반복 실패 케이스를 하네스 업데이트에 반영 |
  | 고객 지원 봇 | 오답 응답을 하네스 프리미티브 수정으로 피드백 |

  도입 난이도: 🟡 중간 (하네스 추상화 설계 필요) / 도입 시기: 2026 Q3~Q4 (코드 공개 후)

- **한계**: 실행 추적 수집 비용이 크고, AEGIS 수렴 속도가 도메인별로 상이함
- **링크**: [https://arxiv.org/abs/2606.14249](https://arxiv.org/abs/2606.14249)

## [2] APPO: Agentic Procedural Policy Optimization

- **저자/소속** 다수, **arXiv** 2606.12384, **카테고리** cs.LG / cs.AI
- **핵심 기여**: "영향력 있는 결정 지점이 도구 호출 경계가 아니라 시퀀스 전체에 분산돼 있다"는 사실을 실증하고, 이를 정밀하게 포착하는 절차(procedure) 수준 크레딧 할당 방법 제안. 기존 agentic RL 기저에 적용하는 드롭인 방식.
- **방법론 요약**: 토큰 불확실성과 이후 시퀀스의 정책 유도 우도 이득을 결합한 **Branching Score**로 고임팩트 결정 지점을 식별한다. 식별된 지점에서 분기 롤아웃을 생성하고, **절차 수준 Advantage Scaling**으로 크레딧을 분배한다. 도구 호출 경계라는 거친 휴리스틱 단위에서 벗어나 탐색 효율과 행동 해석 가능성을 동시에 확보한다.
- **주요 결과**: 강력한 에이전트 RL 기저 대비 **13개 벤치마크 평균 +4p** 향상, 도구 호출 효율성 및 해석 가능성 유지
- **🛠 실무 시사점**:

  | 적용 분야 | 기대 효과 |
  |---|---|
  | 에이전트 파인튜닝 파이프라인 | 크레딧 할당 방식 교체만으로 기저 대비 성능 향상 |
  | 멀티스텝 코드 생성 RL | 중간 결정 지점 추적으로 실패 원인 디버깅 용이 |

  도입 난이도: 🟡 중간 (기존 agentic RL 코드에 Branching Score 삽입 필요) / 도입 시기: 2026 Q3

- **한계**: Branching Score 계산 비용 추가 발생; 초장기 시퀀스에서 분기 수 폭증 가능
- **링크**: [https://arxiv.org/abs/2606.12384](https://arxiv.org/abs/2606.12384)

## [3] Orchestra-o1: Omnimodal Agent Orchestration

- **핵심 기여**: 텍스트·이미지·오디오·비디오를 동시 처리하는 이종(heterogeneous) 에이전트를 모달리티 인식 태스크 분해, 온라인 서브에이전트 전문화, 병렬 실행으로 통합 오케스트레이션하는 프레임워크. DA-GRPO로 학습된 8B 오픈소스 모델 공개.
- **방법론 요약**: 입력 모달리티를 인식해 태스크를 분해한 후, 온라인으로 서브에이전트를 전문화하는 통합 메커니즘을 설계한다. 오케스트레이터 학습에는 결정 정렬 그룹 상대 정책 최적화(**DA-GRPO**)를 적용하며, 병렬 실행 아키텍처로 멀티모달 복잡 태스크의 지연을 최소화한다.
- **주요 결과**: OmniGAIA 벤치마크 **2위 대비 +10.3% 정확도**, 오픈소스 옴니모달 에이전트 중 최고 성능
- **🛠 실무 시사점**:

  | 영상 품질 자동 검사 | 시각·텍스트 기준을 병렬 에이전트로 동시 평가 |
  | 교육 콘텐츠 QA | 강의 영상 + 교재 문서를 동시 분석 |

  도입 난이도: 🔴 높음 (8B 모델 서빙 인프라 및 모달리티별 커넥터 필요) / 도입 시기: 2026 Q4~

- **링크**: [https://arxiv.org/abs/2606.13707](https://arxiv.org/abs/2606.13707)

## [4] Memory is Reconstructed, Not Retrieved: Graph Memory for LLM Agents (MRAgent)

- **저자/소속** National University of Singapore 등, **arXiv** 2606.06036, **카테고리** cs.AI / cs.CL — **ICML 2026 채택** (HuggingFace Daily Papers 오늘 특집)
- **핵심 기여**: 기존 "검색 후 추론(retrieve-then-reason)" 방식이 추론 중 동적 적응을 못 한다는 한계를 지적하고, 메모리 접근을 LLM 추론이 통합된 능동적 **재구성(reconstruction)** 과정으로 재정의하는 MRAgent 제안
- **방법론 요약**: `Cue-Tag-Content` 그래프로 메모리를 구조화하며, 연상 태그(associative tags)가 세부 단서와 콘텐츠를 의미론적으로 연결한다. 추론 중 LLM이 증거를 따라 탐색 경로를 반복적으로 정제하여, 비제한 경로 확장의 조합 폭발 없이 동적 메모리 접근을 구현한다.
- **주요 결과**: LoCoMo·LongMemEval 벤치마크에서 기저 대비 최대 **+23%** 향상, 토큰 소비 및 런타임 비용 동시 절감
- **🛠 실무 시사점**:

  | 적용 분야 | 기대 효과 |
  |---|---|
  | 장기 고객 상담 봇 | 과거 대화 이력을 그래프로 재구성해 맥락 정확도 향상 |
  | 개인 비서 에이전트 | 회의·이메일 이력을 능동적 메모리로 재구성 |
  | 법률·의료 RAG | 근거 추적 가능한 연상 태그로 인용 신뢰도 향상 |

  도입 난이도: 🟡 중간 (`Cue-Tag-Content` 그래프 구조 구현 필요) / 도입 시기: 2026 Q3 (ICML 오픈소스 공개 후)

- **한계**: 그래프 구축 초기 비용 존재; 태그 품질이 검색 정밀도에 직결됨
- **링크**: [https://arxiv.org/abs/2606.06036](https://arxiv.org/abs/2606.06036)

## [5] From AGI to ASI (Google DeepMind)

- **저자/소속** Tim Genewein, Shane Legg, Thore Graepel 등 14인 — **Google DeepMind**, **arXiv** 2606.12683, **카테고리** cs.AI
- **핵심 기여**: "AGI 도입으로 인한 단일 불연속 전환점"이라는 통념에 반론하며, AGI→인공 초지능(ASI) 전환의 4가지 경로를 구조화한 기술 보고서. DeepMind 공동 창업자 Shane Legg가 공저자로 주목.
- **방법론 요약**: 이론적 포지션 페이퍼로, 경로 1(기존 AGI 시스템 확장), 경로 2(AI 패러다임 전환), 경로 3(재귀적 자기개선), 경로 4(대규모 멀티에이전트 집합에서의 창발) 등 네 가지 시나리오를 분석한다. 연속 가속 시나리오를 배제하지 않으면서, 과학·기술 다영역에서 "일련의 변혁적 사회 변화"가 단일 특이점보다 현실적이라는 입장을 취한다. Universal AI 이론을 토대로 병목·마찰 요인도 체계적으로 정리.
- **주요 결과**: 경험적 벤치마크 없음 (이론 분석). 핵심 시사점: AI가 여러 과학·기술 분야에서 변혁적 변화를 일으키는 "점진적 ASI 전환"이 더 개연성 높음
- **🛠 실무 시사점**:

  | 영역 | 시사점 |
  |---|---|
  | 스타트업 전략 | ASI 전환 경로별 투자·제품 시나리오 플래닝 |
  | AI 거버넌스 | 마찰·병목 구간에서 정책 대응 선제 설계 |
  | R&D 로드맵 | 재귀적 자기개선 및 멀티에이전트 창발 경로 모니터링 |

  도입 난이도: 해당 없음 / 도입 시기: 전략 논의에 즉시 활용 가능

- **한계**: 정성적 분석이며, 전환 시기를 특정하지 않아 실무 적용 시 별도 시나리오 플래닝 필요
- **링크**: [https://arxiv.org/abs/2606.12683](https://arxiv.org/abs/2606.12683)

## [1] Tool(param:value) 권한 문법 — 파라미터 기반 세밀한 퍼미션 제어 (CC v2.1.178)

- **출처/링크**: [Claude Code v2.1.178 릴리즈 노트](https://github.com/anthropics/claude-code/releases/tag/v2.1.178), **도구** 🟦CC, **카테고리** ⚙️Workflow
- **💡 핵심 아이디어**: v2.1.178부터 퍼미션 규칙에 `Tool(param:value)` 문법이 추가됐다. 도구 이름이 아니라 **입력 파라미터 값**으로 허용/차단 범위를 지정할 수 있어, 고비용 Opus 모델 서브에이전트만 선택적으로 차단하거나 위험 명령어 패턴을 실행 전 방어하는 정밀 제어가 가능하다. 와일드카드(`*`) 지원으로 패밀리 전체를 한 줄로 처리할 수 있다.
- **📝 구체적 사용법**:

```jsonc
// .claude/settings.json
{
  "permissions": {
    "deny": [
      "Agent(model:claude-opus*)",
      "Bash(command:rm -rf*)",
      "Bash(command:git push --force*)"
    ],
    "allow": [
      "Agent(model:claude-sonnet*)"
    ]
  }
}
```

- **🎯 유용한 상황**: 엔터프라이즈 환경에서 비용 제어(Opus 서브에이전트 차단), CI/CD 파이프라인에서 위험 명령어 사전 차단, 팀별 모델 등급 접근 제한
- **⚠️ 주의/한계**: `deny` 규칙이 `allow`보다 우선 적용됨; 파라미터 이름은 도구 공식 스펙과 정확히 일치해야 하며 오타 시 규칙이 무시됨
- **⭐ 추천도**: ★★★★★ — 권한 관리가 필요한 모든 팀 환경에서 필수

## [2] 중첩 .claude/skills 자동 로드 — 모노레포 패키지별 스킬 격리 (CC v2.1.178)

- **출처/링크**: [Claude Code v2.1.178 릴리즈 노트](https://github.com/anthropics/claude-code/releases/tag/v2.1.178), **도구** 🟦CC, **카테고리** 🎨Skill
- **💡 핵심 아이디어**: 작업 중인 파일의 하위 디렉터리 `.claude/skills`가 자동으로 로드된다. 이름 충돌 시 `<dir>:<skill>` 형식으로 양쪽 접근 가능하며, `agent`·`workflow`·`output-style` 설정은 작업 디렉터리와 가장 가까운 것이 우선 적용된다.
- **📝 구체적 사용법**:

```text
my-monorepo/
├── .claude/skills/
│   └── deploy.md            # 루트 레벨: /deploy
├── packages/
│   └── api/
│       └── .claude/skills/
│           └── deploy.md    # api/ 작업 시 /api:deploy 로 자동 활성화
└── apps/
    └── web/
        └── .claude/skills/
            └── deploy.md    # web/ 작업 시 /web:deploy 로 자동 활성화
```

  `api/src/` 파일을 열고 있을 때 `/api:deploy` 스킬이 자동 제공되며, 루트 `/deploy`와 공존한다.

- **🎯 유용한 상황**: 모노레포에서 패키지마다 다른 배포·테스트·린트 워크플로우를 분리해야 할 때; 여러 팀이 공유 레포에서 팀별 스킬을 격리할 때
- **⚠️ 주의/한계**: 비대화형(`--non-interactive`) 실행 시 기존에 권한 프롬프트가 뜨던 버그는 이번 릴리즈에서 수정됨; 너무 많은 중첩 스킬은 로드 시간 증가 유발 가능
- **⭐ 추천도**: ★★★★☆ — 모노레포 팀에 강력 추천

## [3] Auto Mode 보안 강화 — 서브에이전트 실행 전 분류기 사전 심사 (CC v2.1.178)

- **출처/링크**: [Claude Code v2.1.178 릴리즈 노트](https://github.com/anthropics/claude-code/releases/tag/v2.1.178), **도구** 🟦CC, **카테고리** 🤖Subagent
- **💡 핵심 아이디어**: Auto Mode에서 서브에이전트가 **실행되기 전** 보안 분류기가 해당 요청을 먼저 평가한다. 이전에는 서브에이전트가 차단 대상 작업을 요청해도 검토 없이 실행됐던 보안 공백이 해소됐다. 이 변경과 함께 MCP 서버 수준 `disallowedTools`가 서브에이전트에서 무시되던 버그도 함께 수정됐다.
- **📝 구체적 사용법**:

```bash
# Auto Mode 활성화 — v2.1.178 업데이트 후 자동 적용, 별도 설정 불필요
claude --auto-mode

# 정책 위반 서브에이전트 요청은 실행 전 차단되어 세션 로그에 기록됨
# disallowedTools를 MCP 서버 수준에서도 이제 서브에이전트가 준수함
```

- **🎯 유용한 상황**: 멀티에이전트 파이프라인에서 서브에이전트의 권한 우회 방지; 엔터프라이즈 AI 거버넌스 요건 충족; 규정 준수가 필요한 의료·금융 환경
- **⚠️ 주의/한계**: 분류기 판단이 과도하게 엄격할 경우 정상 작업도 차단될 수 있으므로 초기 파이프라인 테스트 필수
- **⭐ 추천도**: ★★★★☆ — Auto Mode를 사용하는 모든 팀에 업데이트 즉시 적용 권장

## [4] Codex CLI v0.140.0 — /import·/usage·세션 삭제 신기능

- **출처/링크**: [Codex CLI GitHub Releases](https://github.com/openai/codex/releases), **도구** 🟩Codex, **카테고리** ⚙️Workflow
- **💡 핵심 아이디어**: Codex CLI v0.140.0(2026-06-15 릴리즈)은 세 가지 주요 기능을 추가한다. (1) `/import`로 Claude Code 설정을 선택적으로 이식, (2) `/usage`로 일별·주별·누적 토큰 활동 추적, (3) `codex delete`·`/delete`로 세션 안전 삭제. Amazon Bedrock API 키 암호화 로컬 저장도 새로 지원한다.
- **📝 구체적 사용법**:

```bash
# Codex CLI에서 Claude Code 설정 선택 이식
codex
/import
# 가져올 항목 선택 후 확인

# 토큰 사용량 확인 (일별 / 주별 / 누적)
/usage

# 특정 세션 삭제 (확인 단계 포함)
codex delete <session-id>

# 세션 내에서 현재 세션 삭제
/delete
```

- **🎯 유용한 상황**: Claude Code에서 Codex로 전환·병행 시 CLAUDE.md·스킬 설정 이식; 월별 API 비용 모니터링; 오래된 세션 정리로 로컬 저장 공간 관리
- **⚠️ 주의/한계**: `/import`는 선택적이므로 이식할 항목을 직접 지정해야 함; Bedrock 인증은 별도 AWS 자격증명 설정 필요; 세션 삭제는 복구 불가
- **⭐ 추천도**: ★★★★☆ — Claude Code와 Codex를 병행 운용하는 팀에 `/import`가 특히 유용
