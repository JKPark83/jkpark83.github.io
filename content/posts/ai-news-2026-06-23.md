# 📚 주목 논문

오늘은 "에이전트 메모리"를 큰 그림과 실전 기법, 두 각도에서 본다.

### 1) Memory for Autonomous LLM Agents (서베이)
- 저자: Pengfei Du (Hong Kong Research Institute of Technology)
- arXiv: **2603.07670** · 분야: LLM 에이전트 메모리 서베이 (2022–2026 초 정리)
- 핵심: 메모리를 **write–manage–read 루프**로 정의하고, 시간 범위·표현 형식·제어 정책의 3차원 분류 체계를 제시.
- 다섯 메커니즘: ① 컨텍스트 내 압축 ② 검색 기반 저장소(RAG) ③ 반성적 자기개선 ④ 계층적 가상 컨텍스트 ⑤ 정책 학습형 관리.
- 🛠 시사점: 내 에이전트의 메모리가 이 5개 중 어디에 속하는지부터 분류해 보면 빈틈이 보인다. 대부분은 ①+②만 쓰고 ③④⑤를 놓친다.
- ⚠ 한계: 서베이라 직접 벤치마크 수치는 없음.
- 링크: https://arxiv.org/abs/2603.07670

### 2) Active Context Compression: Focus (기법)
- arXiv: **2601.07190** · 주제: 장기 SWE 작업의 "Context Bloat" 완화
- 핵심: ReAct 루프에 `start_focus` / `complete_focus` 두 프리미티브를 추가. 에이전트가 스스로 하위 작업의 학습을 **Knowledge 블록**으로 응축하고 원본 히스토리는 잘라낸다(prune).
- 결과: SWE-bench Lite의 컨텍스트 집약 5건을 Claude Haiku 4.5로 실험, **세션 내 22.7% 토큰 절감**.
- 🛠 시사점: 메모리 서버 없이 프롬프트만으로 적용 가능한 패턴. 긴 디버깅 루프에 바로 이식 가능(아래 공통 팁 참고).
- ⚠ 한계: **N=5 소규모** 사례 연구. 정확도 영향은 더 큰 규모 검증이 필요.
- 링크: https://arxiv.org/abs/2601.07190

![Focus 패턴의 start_focus→explore→complete_focus→prune 흐름과 22.7% 절감 결과 다이어그램](/images/ai-news-2026-06-23/focus-loop.svg)
*Focus 패턴: 하위 작업이 끝날 때마다 학습을 요약해 남기고 원본 로그는 폐기한다.*

# 🧑‍💻 Codex & Claude Code 실전 팁

오늘은 세 팁 모두 "메모리/컨텍스트"로 통일.

### 🟦 Claude Code · Context — Auto memory를 길들이기
- 출처: https://code.claude.com/docs/en/memory
- 💡 핵심: v2.1.59+는 내 수정·교정을 보고 Claude가 스스로 `MEMORY.md`에 노트를 남긴다(기본 ON). CLAUDE.md(내가 쓰는 규칙)와 역할이 다르다.
- 📝 how-to:
  ```bash
  # 세션 안에서 무엇이 저장됐는지 확인·편집
  /memory
  # 끄려면 settings.json
  # { "autoMemoryEnabled": false }
  ```
  큰 CLAUDE.md는 `@path` import로 쪼개고, 경로별 규칙은 `.claude/rules/`로 분리하면 컨텍스트가 가벼워진다.
- 🎯 사용처: 같은 교정을 매 세션 반복 입력하고 있을 때.
- ⚠ 주의: Auto memory는 머신 로컬·레포 단위라 클라우드/머신 간 공유 안 됨. 강제해야 할 규칙은 메모리가 아니라 hook으로.
- ⭐ 추천: 팀 규칙은 CLAUDE.md, "내가 발견한 빌드 요령"은 Auto memory로 분리 운영.

### 🟩 Codex · Context — Memories + AGENTS.md 2층 구조
- 출처: https://developers.openai.com/codex/memories
- 💡 핵심: Codex는 정적 지시(AGENTS.md)와 자동 요약(`~/.codex/memories/`)을 **둘 다** 읽는다. 정적층은 내가, 동적층은 Codex가 채운다.
- 📝 how-to:
  ```text
  # AGENTS.md 탐색·병합 규칙
  # 루트→현재 디렉터리 순으로 이어붙이고, 가까운 파일이 우선
  # 인식 파일: AGENTS.override.md > AGENTS.md > TEAM_GUIDE.md > .agents.md
  ```
  중요한 지시가 잘리면 `project_doc_max_bytes`를 올리거나 디렉터리별로 쪼갠다.
- 🎯 사용처: 모노레포에서 하위 모듈마다 규칙이 다를 때.
- ⚠ 주의: 목록에 없는 파일명은 지시로 인식되지 않는다(예: `CONTEXT.md`).
- ⭐ 추천: 공유 규칙은 AGENTS.md로 커밋, 세션 학습은 Memories에 맡기기.

### 🟪 공통 · Context — start_focus/complete_focus 직접 구현
- 출처: https://arxiv.org/abs/2601.07190
- 💡 핵심: 메모리 서버 없이도 위 Focus 패턴을 도구 두 개로 흉내 낼 수 있다.
- 📝 how-to: 에이전트에 다음 규칙을 주면 끝.
  ```text
  1) 하위 작업 시작 시: 무엇을 조사하는지 한 줄로 선언한다.
  2) 끝나면: {시도한 것 / 배운 사실·파일경로·버그 / 결과}만 5줄 요약으로 남긴다.
  3) 요약을 남긴 뒤에는 그 구간의 원본 로그를 더 참조하지 않는다.
  ```
- 🎯 사용처: 10+ 도구 호출이 도는 장기 디버깅·리팩터링 루프.
- ⚠ 주의: 요약이 곧 기억이므로, 요약 품질이 나쁘면 정보 손실. 핵심 식별자(파일·에러 메시지)는 원문 그대로 남길 것.
- ⭐ 추천: 먼저 내장 메모리로 시도하고, 부족할 때 이 패턴 → 그래도 부족하면 mem0/supermemory.
