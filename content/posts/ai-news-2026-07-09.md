# 🔥 오늘의 핵심

OpenAI **GPT-5.6**가 오늘(7/9) 파트너 프리뷰를 넘어 **ChatGPT·Codex·API 전면 공개**됐다. 핵심은 성능이 아니라 **가격/성능 곡선의 이동**이다 — 막내 **Luna가 $1/$6**로 Claude Opus 4.8를 TerminalBench 2.1에서 앞선다(82.5 vs 78.9). 즉 오늘 개발자의 실전 질문은 하나다: **"내 에이전트 파이프라인의 어느 스텝을 더 싼 모델로 내려도 품질이 안 깨지나?"** — 그 답은 감이 아니라 eval로 확정해야 한다.

![2026-07-09 AI 변화 지도: GPT-5.6 공개, Sonnet 5 기본, Gemini 3.5 연기를 개발자 액션으로 연결](/images/ai-news-2026-07-09/change-map.svg)
*오늘의 변화 지도 — 프론티어 3파전이 재편됐고, 개발자 액션은 "eval 재실행 → 티어 라우팅"으로 수렴한다.*

# 🤖 모델 & 릴리즈 레이더

- **GPT-5.6 Sol·Terra·Luna 전면 공개 (7/9).** 3종 패밀리 — Sol(플래그십), Terra(균형), Luna(초저가·초고속). 가격(1M 토큰): **Sol $5/$30, Terra $2.50/$15, Luna $1/$6**. TerminalBench 2.1에서 **Sol Ultra 91.9 · Sol 88.8**로 Mythos 5(88.0)를 넘고, **Luna 82.5·Terra 84.3**이 Opus 4.8(78.9)를 추월했다. 에이전트 빌더 관점: 코딩·터미널 작업 라우팅 후보가 늘었다. 특히 **Luna는 대량 툴콜/분류/요약 스텝을 싸게 처리**하는 워크호스 후보다. ([공개 일정](https://www.neowin.net/news/openai-to-release-gpt-56-sol-terra-and-luna-on-july-9/), [가격](https://www.aipricing.guru/openai-pricing/))

![GPT-5.6 3종과 레퍼런스 모델의 TerminalBench 2.1 점수 및 출력가격 비교](/images/ai-news-2026-07-09/gpt56-bench.svg)
*TerminalBench 2.1 점수(막대) + 출력가격. 파랑=GPT-5.6, 회색=비교군. 벤치·가격 모두 1차·2차 소스에서 확인한 값.*

- **가격/성능 대비의 함의: "무조건 플래그십"의 시대는 끝났다.** Luna가 옛 프론티어(Opus 4.8) 수준을 $1/$6에 낸다는 건, 파이프라인의 80%를 차지하는 단순 스텝을 **10배 이상 싸게** 굴릴 여지가 생겼다는 뜻. 단, 벤치 우위 ≠ 내 태스크 우위 — **자체 eval 없이는 티어 다운을 하지 마라.**
- **Gemini 3.5 Pro는 여전히 7/17 대기.** 2.5 Pro 아키텍처를 폐기하고 재설계 중(2M 컨텍스트 예고). 아직 붙일 물건 아님 — 벤치 시나리오만 미리 준비. ([llm-stats](https://llm-stats.com/llm-updates))

# 📚 주목 논문

오늘도 **arXiv(`export.arxiv.org`)와 Hugging Face Papers가 네트워크 정책으로 차단**돼 arXiv ID·저자·수치를 1차 소스에서 확정할 수 없었다. 제로 팩트 원칙상 미검증 논문은 싣지 않는다.

- 라이브로 직접 확인 권장: [Hugging Face Daily Papers](https://huggingface.co/papers), [Papers With Code](https://paperswithcode.com). 최근 흐름은 **cost-aware routing / model cascade**(쉬운 쿼리는 싼 모델, 어려운 쿼리만 큰 모델)로, 오늘 GPT-5.6 3종 라인업과 정확히 맞물린다. 상위 논문에서 라우팅·캐스케이드 키워드를 직접 훑어보길.

**🟩 Codex — [Workflow]** ([codex releases](https://github.com/openai/codex/releases))
- 💡 **핵심**: 최신 `rust-v0.143.0`(7/8)이 **시스템 프록시를 자동 인식**하고 **원격 플러그인 가시성**을 개선했다.
- 📝 **방법**: 사내 프록시 뒤에서 돌리는 팀은 이제 별도 환경변수 삽질 없이 시스템 프록시 설정을 그대로 탄다. 버전만 올려라.
```bash
codex --version        # rust-v0.143.0 이상 확인
codex --upgrade        # 아니면 업그레이드
```
- 🎯 **언제**: 기업망·VPN 뒤에서 Codex가 네트워크 에러로 죽던 경우.
- ⚠️ **주의**: 프리뷰 GPT-5.6를 Codex에 물릴 땐 조직 접근 권한/쿼터부터 확인.
- ⭐ **추천**: 프록시 환경 팀은 즉시 업그레이드.

**🟦 Claude Code — [Debugging]** ([CHANGELOG](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md))
- 💡 **핵심**: `2.1.205`(7/8)에서 **`/doctor`가 전체 셋업 점검기로 승격**됐다(문제를 진단+수정, `/checkup`는 별칭).
- 📝 **방법**: 세션이 이상하거나 새 머신에서 시작할 때 제일 먼저 실행.
```text
/doctor        # 설정·MCP·플러그인·인증까지 진단하고 고칠 것을 제안
```
- 🎯 **언제**: "왜 안 되지"류 원인 격리, 신규 환경 온보딩.
- ⚠️ **주의**: 같은 버전은 **auto 모드가 미해석 변수에 `rm -rf`를 돌리기 전 먼저 확인**하도록 강화됐다 — 자동화 스크립트의 변수 전개를 점검하라.
- ⭐ **추천**: 온보딩·디버깅 첫 손.

**🟪 공통 — [Context]** ([ponytail](https://github.com/DietrichGebert/ponytail))
- 💡 **핵심**: 플러그인을 안 깔아도, ponytail의 **"재사용 사다리"를 `CLAUDE.md`/`AGENTS.md`에 규칙으로** 박아두면 어느 툴에서든 신규 코드 폭증을 억제한다.
- 📝 **방법**: 아래를 규칙 파일 상단에 추가.
```md
## 코드 작성 전 체크(위→아래 순서로 멈출 것)
1. 이 코드가 꼭 있어야 하나? 아니면 만들지 마라
2. 코드베이스에 이미 있나? → 재사용
3. 표준 라이브러리/플랫폼 기능인가? → 그걸 써라
4. 이미 설치된 의존성으로 되나? → 그걸 써라
5. 한 줄로 되나? → 한 줄
6. 그제서야 최소한만 새로 작성
```
- 🎯 **언제**: 에이전트가 매번 유틸을 새로 짜서 diff가 비대해질 때.
- ⚠️ **주의**: 정확성이 중요한 구간(보안·동시성)은 "최소 코드"가 오히려 위험 — 사다리 5·6단계에 리뷰 게이트를 걸어라.
- ⭐ **추천**: 리뷰 부담이 큰 팀일수록 효과 큼.
