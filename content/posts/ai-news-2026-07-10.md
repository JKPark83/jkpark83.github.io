# 🔥 오늘의 핵심

오늘(7/10)은 **새 프론티어 모델이 아니라 '코딩 에이전트 툴체인'이 움직인 날**이다. Codex CLI가 어제(7/9) **rust-v0.144.0**에서 `writes` 승인 모드를 내놨다 — 읽기 전용 툴은 막힘없이 돌리고 **디스크에 쓰는 순간만 승인**을 받는, 반자동 코딩의 실전 손잡이다. 동시에 Claude Code는 `/config`에 **동적 워크플로 크기**를 노출해 멀티 에이전트 팬아웃(=토큰/비용)을 사용자가 직접 조인다. 그래서 오늘 개발자가 할 일은 하나: **자율성은 올리고 사고 반경은 좁히는** 이 두 손잡이를 각자 파이프라인에 붙여보는 것.

![2026-07-10 AI 변화 지도: Sonnet 5·Codex v0.144·비주얼 빌더를 개발자 액션으로 연결](/images/ai-news-2026-07-10/change-map.svg)
*오늘의 변화 지도 — 모델보다 툴체인이 움직였다. 액션은 "긴 컨텍스트는 Sonnet 5로, 반자동 코딩은 Codex writes 모드로"로 수렴한다.*

# 🤖 모델 & 릴리즈 레이더

- **오늘 브랜드-뉴 프론티어 모델 소식은 조용.** 대신 실전에서 바로 쓰는 값은 **Claude Sonnet 5**다 — Claude Code **기본 모델**이자 **네이티브 1M 토큰 컨텍스트**, 그리고 **8/31까지 $2/$10(1M 토큰) 프로모 가격**. 에이전트 빌더 관점: 지금이 **긴 컨텍스트 잡(코드베이스 전체 리뷰, 대용량 로그 요약)을 싸게 실험할 창**이다. 프로모 마감이 데드라인이니, 1M 컨텍스트 전제 워크플로가 있으면 이번 달에 검증해두자. ([Claude Code 릴리스 노트](https://code.claude.com/docs/en/changelog))
- **GPT-5.6 패밀리는 이제 Amazon Bedrock에서도 정식 식별.** Codex v0.144.0 체인지로그에 "Bedrock 모델 이름이 GPT-5.6 패밀리와 변형을 명확히 식별"이 포함됐다 — 즉 GPT-5.6 계열이 AWS 라우팅 경로에서 잡힌다는 뜻. 멀티클라우드로 티어 라우팅하는 팀은 Bedrock 쪽 모델 ID를 다시 확인할 때. ([codex rust-v0.144.0](https://github.com/openai/codex/releases/tag/rust-v0.144.0))

# 🛠 오픈소스 스포트라이트

오늘의 축: **"코드 없이 에이전트를 조립한다" — 비주얼 빌더가 프론트라인으로.** 두 레포 모두 MCP 서버 배포를 기본 지원한다.

![비주얼 에이전트 빌더 별점 비교 막대 차트](/images/ai-news-2026-07-10/repo-compare.svg)
*별점은 GitHub 레포 페이지 근사값(2026-07-10 확인). 막대 길이는 별점 비례.*

- **[Langflow](https://github.com/langflow-ai/langflow)** · Python · 약 **151k ★**. 시각적 플로우 빌더 + 파이썬 커스터마이즈 + 플레이그라운드 + API/JSON 내보내기 + **MCP 서버 배포**. 에이전트 개발자 활용: 팀 내 비개발자와 **프롬프트·툴 배선 프로토타입**을 빠르게 합의하고, 확정된 플로우만 코드/JSON으로 내려 프로덕션에 이관.
- **[Dify](https://github.com/langgenius/dify)** · TypeScript · 약 **148k ★**. "프로덕션용 에이전틱 워크플로 개발 플랫폼"을 표방. 활용: RAG 파이프라인·툴 오케스트레이션·관측을 한 판에서 구성하고 **API로 노출**해, 자체 백엔드에 붙이는 매니지드 대안으로. Langflow가 '조립·실험' 쪽이라면 Dify는 '운영·배포' 쪽에 무게가 있다.

### 🟩 Codex — writes 승인 모드 (category: Workflow)
- 출처: [openai/codex rust-v0.144.0](https://github.com/openai/codex/releases/tag/rust-v0.144.0) (7/9)
- 💡 **핵심**: 새 `writes` 앱-승인 모드는 **선언된 읽기 전용 액션은 자동 허용**하고, **파일에 쓰는 동작만 승인 프롬프트**를 띄운다. "전부 승인" ↔ "전부 자동" 사이의 실전 중간값.
- 📝 **how-to**: 승인 모드를 `writes`로 선택하면 grep·읽기·조회는 끊김 없이 흐르고, 실제 파일 수정/생성 시점에만 멈춘다. 탐색이 많은 리팩터링·조사 작업에서 클릭 피로가 확 준다.
- 🎯 **언제**: 코드베이스를 훑으며 고치는 반자동 세션 — 읽기는 자유롭게, 디스크 변경만 게이트하고 싶을 때.
- ⚠️ **주의**: "읽기 전용" 선언을 신뢰하는 구조다. 부작용 있는 MCP 툴이 읽기로 분류되면 그냥 통과하니, 서드파티 툴 세트는 직접 점검.
- ⭐ **추천**: 반자동 코딩 기본값으로 승격. 완전 자동 전 단계로 딱 좋다.

### 🟦 Claude Code — 동적 워크플로 크기 (category: Workflow / Context)
- 출처: [Claude Code 릴리스 노트](https://code.claude.com/docs/en/changelog) (2.1.202)
- 💡 **핵심**: `/config`의 **"Dynamic workflow size"**(small/medium/large)로 Claude가 동적 워크플로에서 **몇 개의 에이전트로 팬아웃할지**를 가이드한다. 멀티 에이전트 = 토큰·비용이므로, 이 손잡이가 곧 비용 손잡이.
- 📝 **how-to**:

```
/config
# → Dynamic workflow size 를 small / medium / large 중 선택
# 대규모 리뷰·마이그레이션처럼 팬아웃이 커지기 쉬운 작업은 small 로 시작
```

- 🎯 **언제**: "리뷰/감사/마이그레이션 전체를 봐줘" 같은 광범위 요청이 수십 개 에이전트로 번질 때, 먼저 규모를 낮춰 비용을 예측 가능하게.
- ⚠️ **주의**: 강제 상한이 아니라 **가이드라인**이다. 진짜 하드 상한은 예산(토큰 목표)·`--max-turns`로 걸어야 한다.
- ⭐ **추천**: 비용에 민감한 팀은 기본을 medium 이하로. 필요할 때만 올려라.

### 🟪 공통 — MCP 대화형 인증이 1급 기능으로 (category: MCP)
- 출처: [codex rust-v0.144.0](https://github.com/openai/codex/releases/tag/rust-v0.144.0), [Claude Code 릴리스 노트](https://code.claude.com/docs/en/changelog)
- 💡 **핵심**: 두 툴이 같은 방향으로 갔다 — **Codex**는 "MCP 툴이 실험 옵트인 없이 대화형으로 인증"을 정식화했고, **Claude Code**(2.1.203)는 세션의 추가 작업 디렉터리를 MCP `roots/list`에 노출하고 변경 시 `notifications/roots/list_changed`를 보낸다.
- 📝 **how-to**: OAuth가 필요한 MCP 서버(예: 이슈 트래커·클라우드 콘솔)를 붙일 때 이제 **환경변수·수동 토큰 우회 없이** 대화형 로그인 흐름을 그대로 태울 수 있다. 멀티 루트 워크스페이스라면 CC가 루트 변경을 서버에 자동 통지하는지도 확인.
- 🎯 **언제**: 사내 인증이 걸린 MCP 커넥터를 에이전트에 물릴 때.
- ⚠️ **주의**: 대화형 로그인은 **디바이스-코드 피싱**에 노출될 수 있다(codex도 이번에 관련 경고 문구를 추가). 낯선 로그인 프롬프트의 출처를 반드시 확인.
- ⭐ **추천**: 인증형 MCP를 쓰는 팀은 두 툴 모두 최신 버전으로.
