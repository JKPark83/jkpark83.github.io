# 🔥 오늘의 핵심

오늘 GitHub 트렌딩 상위를 채운 건 **실제로 행동하는 자율 에이전트**다. 취약점을 직접 익스플로잇하는 `strix`, 웹 UI를 자연어로 클릭·입력하는 `page-agent`. 동시에 CC·Codex는 백그라운드 서브에이전트와 MCP tool search로 **문맥 절약 + 실행 게이팅** 방향으로 움직이고 있다. 즉, "에이전트에게 손발을 달아주되 안전핀을 어떻게 걸 것인가"가 오늘의 주제다.

*오늘의 변화 지도 — 모델(Sonnet 5), 행동형 레포(strix·page-agent), 툴체인(백그라운드 서브에이전트·MCP tool search)이 '실행하는 에이전트'로 수렴한다.*

![strix와 page-agent 별점·성격 비교](/images/ai-news-2026-07-06/repo-compare.svg)
*트렌딩 '행동형' 에이전트 2종 비교. 별점은 GitHub API 근사값(2026-07-05 확인).*

- **[usestrix/strix](https://github.com/usestrix/strix)** — Python · 약 **37,008★**. 자율 펜테스트 에이전트. HTTP 프록시·헤드리스 브라우저·터미널·파이썬 익스플로잇 환경을 갖고 코드를 **동적으로 실행**해 취약점(OWASP Top 10 등)을 찾고 PoC로 실증한다. 그래프 기반 멀티 에이전트로 병렬 탐색. 에이전트 개발자 활용: `pipx install strix-agent` 후 CI/PR 스캔으로 붙여 정적 분석의 오탐 없이 실증형 리포트를 뽑는 용도.
- **[alibaba/page-agent](https://github.com/alibaba/page-agent)** — TypeScript · 약 **23,795★**. 페이지 안에서 도는 GUI 에이전트로, 자연어 지시를 받아 DOM을 직접 클릭·입력한다. 활용: 스크린샷→좌표 추론 없이 in-page로 웹 작업을 자동화하는 브라우저형 에이전트의 액션 레이어로 재사용.

# 🧑‍💻 Codex & Claude Code 실전 팁

## 🟦 Claude Code — Subagent / Workflow

- 카테고리: Subagent · 출처: [Claude Code changelog 2.1.198](https://code.claude.com/docs/en/changelog)
- 💡 **핵심**: 2.1.198부터 서브에이전트가 **기본 백그라운드 실행**. 코드 작업을 worktree에서 끝내면 자동으로 commit·push하고 **draft PR까지 열어준 뒤** `Notification` 훅(`agent_completed`/`agent_needs_input`)을 발화한다.

![CC 백그라운드 서브에이전트 PR 흐름](/images/ai-news-2026-07-06/cc-background-flow.svg)
*delegate → 병렬 작업 → 자동 commit·push·draft PR → Notification 훅. 폴링 대신 이벤트로 완료를 받는다.*

- 📝 **How-to**: 긴 작업을 서브에이전트에 넘기고 메인 세션은 설계·리뷰를 계속한다. 완료 알림은 폴링 대신 훅으로 받는다.

```json
// .claude/settings.json — 서브에이전트 완료 시 데스크톱 알림
{
  "hooks": {
    "Notification": [
      { "matcher": "agent_completed",
        "hooks": [{ "type": "command",
          "command": "notify-send 'CC subagent done'" }] }
    ]
  }
}
```

- 🎯 **언제**: 테스트 실행·대규모 리팩터처럼 오래 걸리는 작업을 병렬화하고 싶을 때.
- ⚠️ **주의**: 백그라운드가 draft PR을 자동 오픈하므로, PR 노이즈를 원치 않으면 위임 범위를 좁게 잡을 것. 유휴 백그라운드 셸은 메모리 압박 시 자동 회수(`CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1`로 비활성).
- ⭐ **추천**: 멀티태스킹 워크플로에 강력 추천.

## 🟩 Codex — MCP / Context

- 카테고리: MCP · Context · 출처: [Codex changelog](https://developers.openai.com/codex/changelog)
- 💡 **핵심**: 7월 업데이트로 **MCP 툴이 기본적으로 tool search 방식**으로 동작한다(지원 모델·프로바이더 한정). 모든 MCP 툴 스키마를 프롬프트에 한꺼번에 올리는 대신 **필요할 때 검색**해 로드 → 문맥 절약.
- 📝 **How-to**: MCP 서버를 여러 개 붙여 툴이 수십 개로 불어난 상황에서, tool search가 켜져 있으면 초기 컨텍스트 점유가 줄어든다. 구형 모델·프로바이더 호환은 유지되니, 오래된 프로필에서 로딩이 이상하면 changelog의 해당 토글을 먼저 확인.
- 🎯 **언제**: MCP 툴 수가 많아 컨텍스트가 툴 스키마로 잠식될 때.
- ⚠️ **주의**: 정확한 설정 플래그 명칭은 세션 시점의 공식 changelog 기준으로 확인(여기선 원문 재확인이 막혀 명칭 단정은 보류).
- ⭐ **추천**: MCP 헤비 유저에게 유용.

## 🟪 공통 — Review / Safety

- 카테고리: Review · 출처: [Claude Code changelog 2.1.200](https://code.claude.com/docs/en/changelog)
- 💡 **핵심**: CC 2.1.200부터 기본 권한 모드가 **"Manual"**, `AskUserQuestion`도 기본 자동 진행 안 함. 오늘 다룬 strix·page-agent처럼 **실제로 실행/클릭하는 에이전트**를 붙일 때, 파괴적 액션 앞에 사람 승인 게이트를 두는 방향과 맞물린다.
- 📝 **How-to**: 행동형 에이전트는 (1) 실행 전 승인 게이트, (2) 격리 샌드박스(컨테이너·worktree), (3) 되돌릴 수 없는 액션만 별도 화이트리스트 — 이 3단 게이트를 기본값으로.
- 🎯 **언제**: 셸·브라우저·네트워크에 손대는 에이전트를 CI나 실환경에 연결할 때.
- ⚠️ **주의**: "자동 진행 끔"이 기본이 되면 무인 파이프라인이 멈출 수 있으니, 배치 잡은 별도 무인 모드로 분리.
- ⭐ **추천**: 실행형 에이전트를 다루면 필수.
