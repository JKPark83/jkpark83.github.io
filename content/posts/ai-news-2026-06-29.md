# 🔥 오늘의 핵심

# 🤖 모델 & 릴리즈 레이더

- **Codex 0.143.x (alpha, ~0.143.0-alpha.29 · 6/28)**: 두 가지가 눈에 띈다. ① **인덱스 웹검색 모드** — 라이브 검색은 허용하되 직접 페이지 접근은 *서버가 승인한 URL*로 제한. ② **멀티에이전트 위임** — app-server 클라이언트가 `disabled / explicit-request-only / proactive`를 스레드·턴 단위로 설정. 그 외 `/usage` 리셋 크레딧 사용, `/plugins` 섹션 정리 추가. → 에이전트의 웹 접근면을 좁히면서 위임 정책을 코드로 통제할 수 있다. 단, 아직 alpha. ([releases](https://github.com/openai/codex/releases) · [changelog](https://developers.openai.com/codex/changelog))
- **Claude Code v2.1.195 (6/26)**: 신규 버전 없음(어제와 동일). 후크 매처가 하이픈 식별자(`code-reviewer`, `mcp__brave-search`)에서 부분일치하던 버그를 **정확일치**로 고쳤다. → 후크 설정을 쓰는 팀은 매처를 한 번 재점검할 값어치가 있다. ([changelog](https://code.claude.com/docs/en/changelog))
- **파운데이션 모델**: 오늘은 큰 신규 가중치 공개 없음. Gemini 3.5 Pro는 6월 말까지도 **미출시**(2M 컨텍스트는 구글이 확인, 가격은 미공개 — 추정치 신뢰 금지). 액션은 모델보다 에이전트 하네스/CLI 쪽에 있다.

# 🧑‍💻 Codex & Claude Code 실전 팁

### 🟩 Codex — 인덱스 웹검색 모드로 에이전트 웹 접근면 좁히기 (Workflow / Security)
- 출처: [Codex changelog](https://developers.openai.com/codex/changelog) (0.143 alpha)
- 💡 **핵심**: 에이전트가 인터넷을 "검색"은 하되, 본문 페치는 **서버가 승인한 URL로만** 제한. 임의 URL 페치·프롬프트 인젝션 표면이 줄어든다.
- 📝 **how-to**: 0.143 alpha에서 사용하되, 공유 환경은 stable로 핀 고정:

```bash
# 시험용: 최신 alpha
npm i -g @openai/codex@0.143.0-alpha.29
# 공유 CI: 동작 고정용 stable
npm i -g @openai/codex@0.142.3 && codex --version
```

- 🎯 **언제**: 외부 데이터를 읽되 신뢰 경계를 지켜야 하는 리서치·요약 에이전트.
- ⚠️ **caveat**: alpha 라인은 동작이 자주 바뀐다. 허용목록 자체가 비어 있으면 검색은 되지만 본문 접근이 막혀 빈 결과처럼 보일 수 있으니 승인 URL 정책을 먼저 확인.
- ⭐ **추천**: 신뢰 경계가 중요한 에이전트라면 도입 가치 큼. 프로덕션은 stable 승격까지 대기.

### 🟦 Claude Code — 후크 매처 정확일치 회귀 점검 (Hook)
- 출처: [Claude Code changelog](https://code.claude.com/docs/en/changelog) (v2.1.195)
- 💡 **핵심**: 후크 매처가 하이픈 든 식별자에서 **부분일치**하던 버그가 정확일치로 수정됐다. 예전 동작에 의존하던 매처는 갑자기 "안 걸릴" 수 있다.
- 📝 **how-to**: 하이픈/네임스페이스가 든 매처를 재확인. 부분일치를 의도했다면 명시적으로 나열한다.

```json
// .claude/settings.json — 정확일치 기준으로 재점검
{
  "hooks": {
    "PostToolUse": [
      { "matcher": "mcp__brave-search", "hooks": [{ "type": "command", "command": "..." }] },
      { "matcher": "code-reviewer",     "hooks": [{ "type": "command", "command": "..." }] }
    ]
  }
}
```

- 🎯 **언제**: `code-reviewer`, `mcp__brave-search`처럼 하이픈/이중밑줄이 든 후크 매처를 쓰는 프로젝트.
- ⚠️ **caveat**: 이건 동작 변경(회귀 수정)이라 업데이트 후 "걸리던 후크가 안 걸리는" 증상이 생길 수 있다 — 의심되면 매처를 먼저 본다.
- ⭐ **추천**: v2.1.195로 올렸다면 후크 매처 1회 전수 점검 권장.

### 🟪 공통 — 멀티 에이전트 CLI는 설정 단일 출처로 묶어라 (Workflow)
- 출처: [cc-switch](https://github.com/farion1231/cc-switch) · [Codex 멀티에이전트 위임](https://developers.openai.com/codex/changelog)
- 💡 **핵심**: Claude Code·Codex·Gemini CLI를 동시에 쓰면 API 키·모델 프로바이더·권한이 닷파일마다 흩어져 드리프트가 난다. cc-switch 같은 전환 도구나 단일 설정 소스로 묶어라.
- 📝 **how-to**: 프로필을 명시적으로 분리하고 한 곳에서 전환.

```text
[ ] 각 CLI의 키/모델/권한을 프로필로 분리 (work / personal / ci)
[ ] 전환은 손편집 대신 도구(cc-switch)나 스크립트로
[ ] CI는 stable 버전 + 최소 권한 프로필로 고정
[ ] 위임 정책(Codex: explicit-request-only)을 명시해 자동 위임 폭주 방지
```

- 🎯 **언제**: 2개 이상 에이전트 CLI를 일상적으로 오가는 팀.
- ⚠️ **caveat**: 전환 도구에 키를 모으면 그 도구가 단일 실패점이 된다 — 접근 권한·시크릿 저장 방식을 먼저 확인.
- ⭐ **추천**: CLI 2개 이상이면 사실상 필수. 시작은 "프로필 분리"부터.
