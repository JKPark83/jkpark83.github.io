## 🟦 Claude Code — Workflow / Context

- 출처: [Claude Code changelog 2.1.202](https://code.claude.com/docs/en/changelog) · [dynamic workflows 문서](https://code.claude.com/docs/en/workflows)
- 💡 **핵심**: 2.1.202에 `/config`의 **"Dynamic workflow size"** 설정이 추가됐다. Claude가 동적 워크플로를 짤 때 목표로 삼는 **에이전트 수(small/medium/large)**를 조절한다. 단, 강제 상한이 아니라 **어드바이스** — 특정 프롬프트가 다른 규모를 요구하면 그게 우선한다.
- 📝 **How-to**: fan-out이 토큰 예산을 태울 때 기본 규모를 낮춰 둔다. 워크플로 실행을 추적하려면 `workflow.run_id`·`workflow.name` OTel 속성으로 재구성.

```text
/config
  → Dynamic workflow size: small   # 기본을 보수적으로
# 큰 작업만 프롬프트에서 "많은 에이전트로 병렬화해줘"로 오버라이드
```

- 🎯 **언제**: 서브에이전트 fan-out이 반복적으로 예산을 넘길 때, 팀 공용 세팅으로 규모를 표준화하고 싶을 때.
- ⚠️ **주의**: 런타임 에이전트 캡은 이 설정과 무관하게 계속 적용된다. 어디까지나 "평소 규모" 가이드다.
- ⭐ **추천**: 워크플로를 자주 쓰는 팀이면 기본값 `small`로 시작해 필요 시 프롬프트로 키우는 쪽이 예산 관리에 안전.

## 🟩 Codex — Permissions / Workflow

- 출처: [Codex changelog](https://developers.openai.com/codex/changelog)
- 💡 **핵심**: `/permissions`가 **named permission profiles**를 이해한다 — 프로젝트/모드별 권한 프로파일을 정의해 전환. 동시에 **Goal mode가 정식(GA)**으로 앱·IDE·CLI에서 사용 가능.
- 📝 **How-to**: 안전 모드와 실행 모드를 프로파일로 분리해 두고, 리스크 있는 작업만 명시적으로 승격.

```text
/permissions          # 설정된 커스텀 프로파일 목록 확인
/permissions use safe # 읽기 위주 프로파일로 전환
# 실제 변경이 필요할 때만 act 프로파일로 승격
```

- 🎯 **언제**: 레포마다 허용 범위가 다를 때, 자율 실행(Goal mode)을 켜기 전에 권한 경계를 고정하고 싶을 때.
- ⚠️ **주의**: 프로파일은 config 단위 — 팀 공유 시 어떤 프로파일이 기본인지 문서화해 두자.
- ⭐ **추천**: 자율성이 올라갈수록 권한을 좁게 시작하는 게 정석. `safe` 기본 + 명시적 승격.

## 🟪 공통 — Context (tool search)

- 출처: [composio](https://github.com/ComposioHQ/composio)
- 💡 **핵심**: CC·Codex·composio가 공통으로 **"모든 툴 스키마를 문맥에 로드하지 말고 검색으로 부른다"**로 수렴 중. 툴이 수십 개면 스키마만으로 문맥이 샌다.
- 📝 **How-to**: MCP/툴 레지스트리를 tool search 뒤에 두고, 태스크가 실제로 요구하는 툴만 로드. 위 워크플로 다이어그램의 레버 ②가 이것.
- 🎯 **언제**: MCP 서버·툴 개수가 두 자리로 늘어난 에이전트.
- ⚠️ **주의**: 검색 미스로 필요한 툴이 안 뜨면 실패한다 — 핵심 툴은 이름·설명을 검색 친화적으로.
- ⭐ **추천**: 툴 10개를 넘기면 tool search 전환을 기본값으로.
