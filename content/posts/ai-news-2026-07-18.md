# 🔥 오늘의 핵심

오늘의 흐름은 하나로 모인다: **에이전트가 무한루프로 돈·컨텍스트를 태우는 문제를 툴체인이 정면으로 잡기 시작했다.** Claude Code v2.1.212는 세션당 subagent·WebSearch 상한을, Codex는 rollout 토큰 예산을 넣었고, MCP 2026-07-28 RC는 프로토콜 코어를 stateless로 바꿔 서버 확장을 단순화했다. 지금 당장 할 일: 에이전트 러너에 **상한값·예산 경고**를 붙이고, MCP 서버는 세션 상태 의존을 걷어내는 준비를 시작하라.

# 📚 주목 논문

## 1) SEED: 에이전트 RL의 자기진화 온폴리시 증류
- **저자**: Jinyang Wu, Shuo Yang, Zhengxi Lu 외 (arXiv `2607.14777`, cs.CL, 2026-07-16 제출)
- **핵심**: 에이전트 RL의 "감독 신호 부족" 문제를, 완료된 궤적을 재사용 가능한 **자연어 스킬**로 바꿔 해결한다.
- **방법(요약)**: 성공 궤적에서 재사용 워크플로를 스킬로 추출 → 다른 컨텍스트에서 액션을 재채점(re-score) → 희소하던 보상을 **dense 학습 신호**로 전환.
- **결과**: 텍스트·비전 과제에서 성능과 샘플 효율이 일관되게 향상, 미지 시나리오로의 일반화도 견고(수치 세부는 논문 확인).
- **🛠 agent-builder 시사점**: 성공 트레이스를 버리지 말고 "스킬 라이브러리"로 축적하면, 별도 라벨 없이도 다음 학습·프롬프트의 dense 신호가 된다.
- **한계**: 스킬 품질이 초기 궤적 품질에 의존 — 나쁜 성공(운 좋은 성공)이 스킬로 굳으면 노이즈가 된다.
- 링크: [arxiv.org/abs/2607.14777](https://arxiv.org/abs/2607.14777)

## 2) Beyond Success Rate: 비용을 반영한 보안 에이전트 평가
- **저자**: Paul Kassianik, Blaine Nelson, Yaron Singer (arXiv `2607.15263`, cs.CR, 2026-07-16 제출)
- **핵심**: 최고 성능만 보지 말고 **비용-성공(cost-success) 프레임**으로 공격·방어 보안 에이전트를 평가하자.
- **방법(요약)**: 공격은 Cybench, 방어는 Splunk BOTS로 측정하며 성공률 대비 소요 컴퓨트를 함께 본다.
- **결과**: 공격 과제는 컴퓨트를 더 부으면 개선되지만, 방어 조사는 **절제된 도구 사용·텔레메트리 탐색·선별적 보강**에 더 좌우된다.
- **🛠 agent-builder 시사점**: 에이전트 벤치를 "성공률"만으로 자랑하지 말고, **성공률 ÷ 토큰/시간** 축을 리포트에 넣어라. 오늘의 Codex 토큰 예산과 바로 맞물린다.
- **한계**: 도메인 특화(보안) 결론 — 일반 코딩/리서치 에이전트로의 이전은 별도 검증 필요.
- 링크: [arxiv.org/abs/2607.15263](https://arxiv.org/abs/2607.15263)

# 🛠 오픈소스 스포트라이트

MCP가 stateless로 가는 지금, "MCP-네이티브"로 설계된 에이전트 프레임워크를 미리 봐두면 마이그레이션 부담이 준다.

| 리포 | 언어 | 요약 |
| --- | --- | --- |
| [lastmile-ai/mcp-agent](https://github.com/lastmile-ai/mcp-agent) | Python | MCP + 단순 워크플로 패턴으로 에이전트 구성 |
| [microsoft/agent-framework](https://github.com/microsoft/agent-framework) | C#/Python | Semantic Kernel + AutoGen 통합 1.0 |

- **lastmile-ai/mcp-agent** — MCP 서버들을 조합해 "효과적인 에이전트"를 만드는 경량 패턴 모음. stateless 코어로 넘어갈 때 세션 상태에 덜 묶인 설계를 참고하기 좋다. **쓸 곳**: 여러 MCP 툴을 오케스트레이션하는 얇은 에이전트 러너의 레퍼런스.
- **microsoft/agent-framework** — 2026년 4월 Semantic Kernel과 AutoGen을 하나로 합친 1.0 라인. **쓸 곳**: .NET/파이썬 혼합 환경에서 멀티 에이전트 오케스트레이션 표준을 잡을 때.
- ⚠️ **투명성 노트**: 이번 세션은 GitHub API 접근이 현재 리포로 제한돼 **스타 수를 검증하지 못했다.** 수치를 지어내지 않기 위해 별점은 생략했으니, 채택 전 각 리포에서 직접 확인하길 권한다.

# 🧑‍💻 Codex & Claude Code 실전 팁

### 🟩 Codex — [Context / Cost]
- 출처: [developers.openai.com/codex/changelog](https://developers.openai.com/codex/changelog)
- 💡 **핵심**: 멀티 에이전트 위임을 기본 켜두지 말고 예산과 함께 레벨을 정하라.
- 📝 **방법**: 위임 정책을 워크로드별로 나눈다.

```
# 탐색·리서치: 위임 허용하되 예산 캡
delegation = "proactive"     # 병렬 서브에이전트 허용
rollout_token_budget = 300000  # 소진 시 턴 abort

# 프로덕션 자동수정: 위임 최소화
delegation = "explicit-request-only"
rollout_token_budget = 80000
```

- 🎯 **언제**: 서브에이전트가 조용히 토큰을 태우는 장기 태스크.
- ⚠️ **주의**: 예산이 너무 낮으면 유효한 작업 중간에 abort된다 — 실제 소비 로그를 보고 캡을 잡아라.
- ⭐ **추천**: 워크로드별로 "탐색용/프로덕션용" 두 프로필을 상시 두기.

### 🟦 Claude Code — [Workflow / Guardrail]
- 출처: [code.claude.com/docs/en/changelog](https://code.claude.com/docs/en/changelog)
- 💡 **핵심**: v2.1.212의 세션 상한을 낮춰 CI/무인 실행의 폭주를 막아라.
- 📝 **방법**: 무인 실행 셸에서 상한을 명시한다.

```bash
export CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION=20
export CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION=30
# 긴 MCP 호출은 자동 백그라운드 (임계값 조절)
export CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS=90000
```

- 🎯 **언제**: 사람이 안 보는 스케줄·CI 세션. 기본값 200은 무인 실행엔 너무 헐겁다.
- ⚠️ **주의**: `/clear`는 subagent 예산을 리셋하니, 장기 세션에서 클리어 남발 시 상한 의미가 약해진다.
- ⭐ **추천**: 대화형 세션엔 `/fork`(별도 백그라운드), 인세션 분기엔 `/subtask`로 역할을 나눠라.

### 🟪 공통 — [MCP / Migration]
- 출처: [blog.modelcontextprotocol.io](https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/)
- 💡 **핵심**: 7/28 사양 전에 MCP 서버의 **세션 상태 의존을 걷어내라.**
- 📝 **방법**: `initialize` 핸드셰이크·서버 메모리에 담아둔 상태를 요청 파라미터/외부 저장소로 이전한다. 그러면 어떤 인스턴스가 요청을 받아도 동작해 라운드로빈 LB 뒤에 그냥 놓을 수 있다.
- 🎯 **언제**: MCP 서버를 여러 인스턴스로 스케일아웃하려는 팀.
- ⚠️ **주의**: 지금은 RC — Beta SDK로 검증만 하고, 프로덕션 고정은 정식 사양(7/28) 이후로.
- ⭐ **추천**: 인가는 OAuth 2.1/OIDC 정렬(iss 검증)로 맞춰 두면 이후 마이그레이션이 매끄럽다.
