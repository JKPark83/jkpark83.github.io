# 🔥 오늘의 핵심

오늘의 한 줄: **도구를 하나씩 호출하지 말고 "코드로" 묶어라.** Microsoft가 BUILD 2026에서 `agent-framework`에 **CodeAct**를 넣었다. 모델에게 도구를 하나씩 시키는 대신 `execute_code` 하나만 주고, 모델이 짧은 프로그램을 써서 도구들을 한 번에 오케스트레이션한다. Hyperlight micro-VM에서 격리 실행하며 **지연 ~50%, 토큰 60%+** 절감(공식 대표 워크로드 기준). 이건 작년 11월 Anthropic의 *code execution with MCP* 패턴이 실제 프레임워크로 내려온 사건 — 오늘 할 일은 "내 에이전트의 다단계 도구 호출을 코드 한 블록으로 접을 수 있나" 점검이다.

![도구 호출을 코드로 묶는 Code Mode를 중심으로 Anthropic·Microsoft·오픈소스·Claude Code·Codex를 연결한 변화 지도](/images/ai-news-2026-06-27/change-map.svg)
*오늘의 변화 지도: 가운데 "Code Mode"를 두고 패턴 원조(Anthropic)·신규 구현(Microsoft)·프레임워크·코딩 도구를 연결.*

# 📚 주목 논문

오늘도 검증 가능한 신규 논문을 싣지 못했다. 실행 환경의 egress 정책상 arXiv(`export.arxiv.org`)가 403으로 차단되어 arXiv ID·카테고리·저자를 1차 출처로 **검증할 수 없었다.** 무검증 인용은 zero-fabrication 원칙에 어긋나므로 논문 섹션은 생략한다. 대신 오늘은 "패턴 → 제품화"가 또렷한 Code Mode 흐름에 집중한다.

- 📝 사용법: 온보딩 화면 또는 설정에서 "Migrate to Codex"를 실행해 기존 Claude Code 구성을 불러온다. 가져온 뒤엔 도구별로 갈라지는 설정(권한·MCP 서버·커맨드)을 한 번 diff 떠서 점검.

# 공용/CI 환경이라면 alpha 회귀를 피하려 stable로 고정
codex --version              # 현재 버전 확인 (예: 0.142.3)
# 팀 환경에서는 0.143.0-alpha 대신 0.142.x stable 핀 권장

- 🎯 언제: Claude Code에서 Codex로 옮기거나 둘을 함께 쓰며 설정을 일원화하고 싶을 때.
- ⚠️ 주의: "supported setup"만 이식된다 — MCP 서버·훅·커스텀 커맨드가 100% 1:1로 넘어온다고 가정하지 말 것. 이식 후 실제 동작을 한 번 돌려 검증.
- ⭐ 추천: 두 도구를 오가는 팀이라면 온보딩 때 한 번 돌려두면 설정 표류를 줄인다.

### 🟦 Claude Code — `/rewind`로 `/clear` 이전까지 복구 (Category: Workflow)
- 출처: [Claude Code changelog](https://code.claude.com/docs/en/changelog) (v2.1.191)
- 💡 핵심: 실수로 `/clear`한 뒤에도 **`/rewind`로 그 이전 대화 시점으로 되감기**가 된다. 긴 컨텍스트를 날려 멘붕하던 상황의 안전망.
- 📝 사용법: 컨텍스트가 꼬였을 때 `/rewind`로 체크포인트를 거슬러 올라가 복구 → 거기서 다시 진행. 같은 릴리즈에서 "콤마로 구분된 훅 매처가 조용히 안 걸리던" 버그도 고쳐졌으니, 멀티 매처 훅을 쓰는 사람은 이번에 동작을 재확인.

```text
/rewind        # 체크포인트 목록에서 /clear 이전 시점 선택
```

- 🎯 언제: 무인/장시간 세션에서 컨텍스트를 날렸거나, 잘못된 분기로 들어가 되돌리고 싶을 때.
- ⚠️ 주의: `/rewind`는 대화 상태 복구지, 파일 시스템 자동 롤백이 아니다 — 디스크 변경은 git으로 따로 관리.
- ⭐ 추천: 자동화 파이프라인을 돌리면 훅 매처 수정 + `/rewind` 둘 다 안정성에 직접 기여한다.

### 🟪 공통 — Code Mode: 도구를 "코드로" 호출하기 (Category: MCP / Context)
- 출처: [Anthropic — Code execution with MCP](https://www.anthropic.com/engineering/code-execution-with-mcp), [Microsoft CodeAct](https://devblogs.microsoft.com/agent-framework/codeact-with-hyperlight/)
- 💡 핵심: 도구를 모델이 하나씩 부르면 **중간결과 전체가 매 턴 모델 컨텍스트를 통과**해 토큰이 폭증한다. MCP 도구를 코드 레벨 API로 노출하고, 모델은 그것들을 호출·가공하는 **짧은 프로그램 한 번**만 작성한다. 중간결과는 실행 환경에 머무르고 모델은 **요약만** 본다.

![기존의 매 호출 방식과 CodeAct 단일 execute_code 방식의 토큰 흐름 비교 다이어그램](/images/ai-news-2026-06-27/codeact-flow.svg)
*상단: 도구마다 모델 왕복(컨텍스트 폭증) / 하단: execute_code 1회로 샌드박스에서 묶고 요약만 반환.*

- 📝 사용법(개념 코드): 도구를 직접 호출하는 대신 코드에서 `call_tool`로 묶고, 큰 중간 데이터는 모델로 올리지 말 것.

```python
# 모델이 "도구를 코드로" 부르는 형태 (CodeAct / code-exec-with-MCP 개념)
def run():
    rows = call_tool("drive.fetch", {"file": "transcript"})   # 큰 데이터
    picked = [r for r in rows if r["score"] > 0.8]             # 로컬에서 필터
    call_tool("crm.update", {"items": picked})
    return f"{len(picked)}건 반영"   # 모델에는 요약만 반환
```

- 🎯 언제: 도구 호출이 3단계 이상이거나, 한 도구의 출력이 커서 컨텍스트를 잡아먹을 때(파일 본문·DB 덤프·로그).
- ⚠️ 주의: 모델이 짠 코드를 실행하므로 **격리 샌드박스/권한 화이트리스트가 필수**(Microsoft는 Hyperlight micro-VM로 망·FS 접근을 차단). 인용 수치(~50%/60%+)는 공식 대표 워크로드 기준 근사값이라 워크로드별로 다르다.
- ⭐ 추천: 다단계·대용량 도구 파이프라인이 있다면 오늘 한 흐름만 Code Mode로 바꿔 토큰 곡선을 측정해보라.
