# 🔥 오늘의 핵심

오늘의 한 줄: **모델은 잠잠한데 토큰값이 올랐다.** Codex 쪽에선 6월 중순부터 토큰 단가가 체감 10~20배로 뛰어 5시간 예산이 2~3프롬프트에 마르는 사례 리포트가 올라온다. 그래서 오늘의 할 일은 새 모델 따라가기가 아니라 **에이전트 루프의 토큰 비용을 깎는 것** — 코드 검색을 시맨틱으로 바꾸고(semble), 반복 작업을 스킬로 증류하고(SkillOpt), 런타임에 토큰 예산을 강제(Codex)하는 세 가지 레버.

![에이전트 토큰 효율을 중심으로 semble·SkillOpt·Codex·Claude Code를 연결한 오늘의 변화 지도](/images/ai-news-2026-06-25/change-map.svg)
*오늘의 변화 지도: 가운데 "토큰 효율 에이전트"를 두고 절약 도구(좌)와 도구 업데이트(우)를 연결.*

# 🤖 모델 & 릴리즈 레이더

오늘은 큰 모델 소식 없음. 대신 코딩 에이전트 도구 쪽 미세 업데이트만.

- **Claude Code v2.1.190** (6/24): 버그픽스·안정화 위주 릴리즈. 전날 v2.1.187에서 들어온 `sandbox.credentials`(자격증명 차단)와 조직 모델 제한이 핵심이었고, 190은 그 위의 안정화. 에이전트 빌더 입장에선 지금이 187 기능을 켜고 굳히기 좋은 타이밍. ([changelog](https://code.claude.com/docs/en/changelog))
- **Codex 0.143.0-alpha** 연속 릴리즈(6/24, alpha.13~15): 정식이 아니라 빠른 알파 사이클. 굵직한 기능은 changelog 쪽에 모임(아래 팁 참고). 알파는 회귀 위험이 있으니 팀 공용 환경은 stable 핀 권장. ([releases](https://github.com/openai/codex/releases))
- **토큰 단가 이슈**: Codex Plus 사용자들이 "6/16 이후 토큰당 rate-limit 비용이 10~20배로 뛰어 5h 예산이 2~3프롬프트에 소진된다"고 보고 중. 단순 가십이 아니라 **에이전트 예산 설계를 바꿔야 할 신호**. ([openai/codex#28879](https://github.com/openai/codex/issues/28879))

# 📚 주목 논문

오늘은 검증 가능한 신규 논문을 싣지 못했다. 실행 환경의 egress 정책상 arXiv(`export.arxiv.org`)와 Hugging Face Papers 접근이 모두 차단되어, arXiv ID·카테고리·저자를 1차 출처로 **검증할 수 없었다.** 무검증 인용은 zero-fabrication 원칙에 어긋나므로 논문 섹션은 생략한다. 내일 접근이 풀리면 다시 채운다.

# 🛠 오픈소스 스포트라이트

토큰 비용이 오른 날, 두 저장소가 정확히 그 문제를 친다. 둘 다 검증된 GitHub 수치.

![SkillOpt 9137스타와 semble 5383스타를 비교한 막대 차트](/images/ai-news-2026-06-25/repo-compare.svg)
*스타 수 비교(GitHub API 검증값). 둘 다 Python·MIT 라이선스.*

- **[MinishLab/semble](https://github.com/MinishLab/semble)** · Python · 약 5,383★ · MIT
  - 에이전트용 **코드 검색 MCP 서버**. 시맨틱 임베딩으로 관련 스니펫만 돌려줘 `grep + read` 방식 대비 **토큰을 약 98% 줄인다**(저장소 자체 설명).
  - 활용: 코드베이스가 큰 SWE 에이전트에서 검색 툴을 semble로 교체하면, 한 턴에 파일 통째로 읽어 컨텍스트를 태우는 패턴을 곧장 차단. MCP라 Claude Code·Codex에 바로 붙는다.
- **[microsoft/SkillOpt](https://github.com/microsoft/SkillOpt)** · Python · 약 9,137★ · MIT
  - **frozen LLM 에이전트용 스킬 자동 최적화**. trajectory를 보고 자연어 스킬을 편집·검증·갱신해 배포 가능한 `best_skill.md` 아티팩트로 떨군다.
  - 활용: 파인튜닝 없이 "자주 하는 작업"을 스킬로 증류 → 매번 긴 프롬프트로 풀어 설명하던 토큰을 한 번 만든 스킬 파일로 대체. self-evolving 에이전트의 가벼운 시작점.

# 🧑‍💻 Codex & Claude Code 실전 팁

오늘 주제는 일관되게 **"토큰을 어디서 깎나"**.

### 🟩 Codex — 롤아웃 토큰 예산 (Category: Workflow)
- 출처: [Codex changelog](https://developers.openai.com/codex/changelog), 관련 PR `#28746`
- 💡 핵심: 에이전트 스레드 전반의 토큰 사용을 추적하고, 남은 예산을 리마인드하다가 **소진 시 턴을 자동 중단**. 토큰 단가가 뛴 지금 폭주 방지에 직결.
- 📝 사용법: `/usage`로 일·주·누적 토큰 활동을 보고, 롤아웃 토큰 예산을 설정해 둔다. 예산이 바닥나면 그 턴을 끊어 무한 도구 루프가 비용을 태우는 사고를 막는다.

```bash
# 현재 사용량/리셋 크레딧 확인 → 예산 감각 잡기
/usage
# (그 외) 적립된 rate-limit 리셋 크레딧을 확인·사용 가능
```

- 🎯 언제: 장시간 도는 멀티턴 에이전트, CI/원격 러너에서 토큰 폭주가 무서운 경우.
- ⚠️ 주의: 예산 중단은 "안전장치"지 "정답"이 아님 — 턴이 잘려 작업이 중간에 멈출 수 있으니 재개 전략을 함께 둘 것.
- ⭐ 추천: 6/16 이후 단가 급등을 겪었다면 오늘 바로 예산을 켜라.

### 🟦 Claude Code — semble를 MCP 검색 서버로 (Category: MCP)
- 출처: [MinishLab/semble](https://github.com/MinishLab/semble), [claude mcp 문서](https://code.claude.com/docs/en/mcp)
- 💡 핵심: 에이전트가 코드를 찾을 때 `grep`으로 훑고 매칭 파일을 통째 `read`하면 컨텍스트가 폭증한다. semble를 MCP로 붙여 **관련 스니펫만** 받게 바꾼다.

![grep+read 방식과 semble MCP 방식의 토큰 흐름 비교 다이어그램](/images/ai-news-2026-06-25/token-flow.svg)
*기존 grep+read(상단, 컨텍스트 폭증)와 semble MCP(하단, 토큰 ~98%↓) 흐름 비교.*

- 📝 사용법:

```bash
# semble를 MCP 서버로 등록 (저장소 README의 실행 방식에 맞춰 command 지정)
claude mcp add semble -- <semble-mcp-실행-커맨드>
claude mcp list   # 등록 확인
```

- 🎯 언제: 파일 수가 많은 모노레포에서 에이전트가 "어디 있지"를 반복 검색할 때.
- ⚠️ 주의: 임베딩 인덱싱 비용·신선도(인덱스 갱신 주기)를 확인. 수치 98%는 저장소 주장이며 코드베이스·쿼리에 따라 다르다.
- ⭐ 추천: grep 의존 에이전트라면 검색 툴만 바꿔도 토큰 곡선이 눈에 띄게 꺾인다.

### 🟪 공통 — 반복 작업은 스킬로 증류 (Category: Context)
- 출처: [microsoft/SkillOpt](https://github.com/microsoft/SkillOpt)
- 💡 핵심: 같은 절차를 매 턴 장문 프롬프트로 재설명하면 그게 곧 토큰 낭비. 잘 도는 trajectory를 **한 번 스킬 파일로 굳혀** 재사용한다(`best_skill.md` 사고방식).
- 📝 사용법: 성공한 작업 로그에서 "무엇을·어떤 순서로·무슨 검증으로" 했는지를 5~15줄로 추출 → 스킬/`CLAUDE.md`·`AGENTS.md`에 고정 → 다음부터는 길게 설명 대신 그 스킬을 참조.
- 🎯 언제: 배포·리팩터·릴리즈 점검처럼 매번 비슷하게 반복되는 절차.
- ⚠️ 주의: 스킬도 오래되면 거짓 안내가 된다 — 검증 게이트(통과해야 갱신)를 두고 주기적으로 재검증.
- ⭐ 추천: SkillOpt를 안 써도 "수동 증류"부터 시작하면 효과 즉시.
